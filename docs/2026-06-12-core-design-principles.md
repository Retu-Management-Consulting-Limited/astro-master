## 1. 收割飞轮（五段闭环）

```
①制造「被看穿」→ ②一刀一糖 → ③背后偷偷变准 → ④删不掉 → ⑤情绪溢出=传播 → 拉新回到①
```

| 段 | 动作 | 机关 |
|---|---|---|
| **① 制造「被看穿」** | 说中她从没告诉任何人的那句 | 一身冷汗「它怎么知道」＝那口多巴胺 |
| **② 一刀一糖** | 戳痛 → 立刻给「原来我没问题」的释放 | 准得让她哭，暖得让她离不开 |
| **③ 背后偷偷变准** | 每次反馈、每次吐槽都在喂模型 | 三个月后它比她闺蜜还懂她 |
| **④ 删不掉** | 删它＝杀掉「唯一懂我的那个」 | 护城河是她在你这里被养出来的那个自己 |
| **⑤ 情绪溢出=传播** | 在情绪峰值那一秒把分享塞她手里 | 一次溢出＝一个新猎物，回到 ① |

## 2. 六条操作原则（每条是一件武器）

### 原则 1 · 不卖「准」，卖「被看穿」
没人因为你预测对未来而上瘾。她上瘾，是因为某句话说中了她从没对任何人讲过的事——那一身冷汗「它怎么知道」就是那口糖。别费劲去算命，去**制造「被看穿」的瞬间，越密越好**。表面占星，底层是它在偷偷给她建模：喂得越多，戳得越准。

### 原则 2 · 戳痛是钩，戳完必须塞一口糖
纯戳痛＝她跑。戳完那一下要**立刻**给「原来我没问题、原来我天生这样」的释放。一刀一糖的节奏才出多巴胺。恐惧让人逃，被理解让人半夜还想再开一次。要狠就狠在这——**准得让她哭，暖得让她离不开**。

### 原则 3 · 真正的机关：它在背后偷偷变准
她每按一次「准/不准」、每吐槽一次前任和老板，模型都在悄悄收窄。三个月后它比她闺蜜还懂她。这就是她删不掉的原因——删了等于亲手杀掉「唯一懂我的那个」。**护城河不是算法，是她在你这里被养出来的那个自己。**

### 原则 4 · 暗箱要藏死
她永远只能看到「大师好懂我」，绝不能看到「它在给我建档」。一旦她感觉被监视、被算计，魔法秒变恐怖。所有偷偷收割，都得裹在「它在为我好」的糖衣里——**每偷一个信号，都要当场还她一个好处**。

### 原则 5 · 传播不是加按钮，是接住情绪溢出
准到起鸡皮疙瘩→她憋不住截图丢群；合盘→她必须把那个人拉进来；身分卡→她要晒「这就是我」。你要做的不是「分享功能」，是在她情绪冲到顶的那一秒，**把分享塞进她手里**。一次溢出＝一个新猎物。

### 原则 6 · 红线只有一条：别把人戳到崩溃
不讲道德，讲猎人守则：把人越戳越焦虑，DAU 短期会涨，但她会恨你、骂你、卸载你，还反向到处说「这 app 让我更焦虑」。这叫**杀鸡取卵**。老猎人要的是她一辈子离不开，不是爽一周然后反咬。所以「接住」不是发善心，是**唯一能让收割持续下去的技术动作**。

## 4. 设计不变量 & 自查清单（硬约束）

> 缘起：设计「每日仪式」时，曾把"准不准"的验证放进「今天」——但未结束的一天无法被验证。验证只能回溯。此类**时间/因果方向错误**，靠下列不变量在出图前拦截。

### 4.1 时间/因果不变量

- **验证永远回溯，预测永远前瞻。** 任何"是不是/准不准/说中了吗"的反馈，其指向的事件必须**已发生、且用户此刻可观测**。未结束或未来之事不可被要求验证。
- **「昨天/今天/明天」是功能三分，不是时间装饰**：昨天＝验证（喂模型+坐实准）｜今天＝指引（前瞻可行动）｜明天＝钩子（留存）。今天的指引，明天自动成为被验证的「昨天」。
- **具体性给内在、谦抑给外部**——连验证问句也守：问「昨天你是不是特别想抓住一个确定答案」（内在、易命中、可证伪风险低），不问「昨天你是不是收到好消息」（外部、易落空、咒语一破即失）。
- **相位有 orb 与快慢**：快图层（月亮/内行星，日内—数日）撑日常循环；慢图层（外行星，数月—数年）撑章节大叙事。两层不可混用于同一时间尺度的内容。

### 4.2 出图前自查清单（每个屏/流必过）

1. **时间方向**：每个"反馈/验证"指向的事，是否已发生且用户可观测？（否 → 改为回溯）
2. **领域机制对照**：本设计是否符合占星真实的时间/因果/数学约束？（预测前瞻、验证回溯、orb、快慢分层）
3. **流畅黄灯**：这个流是否"看起来太顺"？越完整越要抽查一条承重的机制假设。
4. **承重假设单验**：本单元里若某条假设错了会整块崩，是否已单独验证它？

## 5. 平台与市场不变量：H5 Mobile Web App × 海外华人 × 双语

> 产品形态固定为基于 H5 的移动端 web app（PWA 增强），**目标市场为海外华人**（非 mainland、不依赖微信服务号生态）。这不是纯技术选型，它改写若干设计决策。

### 5.1 H5 强化的（与增长论天生契合）

- **零安装 · 链接即传播**：合盘邀请、金句卡、一键转发是病毒飞轮⑤的核心，H5 省掉应用商店安装墙，且海外分享无微信 webview 限制（小红书/IG/TikTok/WhatsApp/Telegram 通吃）。
- **全球化登录**：Google / Apple / Email 一键登录，近零注册门槛，强化「零注册先体验」。
- **Stripe 订阅（USD）**（变现期）：仍规避苹果 30% 抽成，海外华人 WTP 更高。
- **快速迭代 · 免 ICP 备案**：Vercel 全球部署，发版即生效，正好契合现有 Next.js + Vercel 工具链。

### 5.2 H5 强制改写的（机制冲突）

- **无可靠原生 push** → **每日召回管道**＝**Email（跨平台可靠兜底）+ Web Push（安卓浏览器直接可用；iOS 需 PWA 加桌，16.4+）+ PWA 加桌（本身即留存提升，并解锁 iOS push）**。「昨/今/明」日循环的设计不变，仅投递管道改变。
  - *注：本条曾误设为「微信服务号订阅消息」，随市场转向海外华人而推翻。*
- **首屏性能严苛**：认知/考虑/转化阶段不等人；盘面用轻量 SVG，控制首屏体积。

### 5.3 海外华人专属痛（差异化金矿，mainland app 未覆盖）

在四大痛域之上叠加离散族群特有的痛：**文化夹缝/身份认同、孤独与疏离、移民/签证/异乡职涯（叠加 AI 时代）、远距关系/离家**。占星＝离散族群的「意义锚」，「被看见」对漂在外面的人力道更猛。

### 5.4 语言：双语定位（中 / 英）

- **产品从一开始即双语（中文 + English），i18n 内建**，不走「mainland 先行、海外后补 pass」。中文为 beachhead（小红书海外华人女性）默认，English 就绪以备国际化扩张（Iris 名字本身即为此选）。
- **Iris 的声音须中 / 英双语原生撰写，非翻译**：C 毒舌通透的语感、金句在两种语言里是不同 idiom（如「不用翻译自己」是中文专属，English 金句须另作），否则英文会有「翻译腔」、毒舌变生硬。
- 配套「旅程地图」「人物形象」中的 mainland 风味细节（¥、城市、微信）视为示意，统一读作 **海外 / 双语 / USD / 小红书·IG** 口径，不另做 pass。
