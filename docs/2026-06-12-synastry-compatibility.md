# 星盘大师 · 传播机关之二：合盘（Synastry）

> **文档类型**：功能设计 / 传播机制（K 因子引擎）
> **状态**：设计进行中（brainstorm 阶段产出）
> **日期**：2026-06-12
> **配套**：[核心设计原则](2026-06-12-core-design-principles.md) · [金句卡](2026-06-12-viral-share-card.md)
> **对应原则**：飞轮⑤「情绪溢出＝传播」；合盘是增长模型中唯一能让 **K 因子 > 1** 的机关

---

## 1. 定位：强制双人 vs 金句卡的广播

- 金句卡＝一对多广播（被动曝光）。
- 合盘＝强制把第二个人拉进场。每次合盘 = 一次潜在新用户获取，且新人自己也上钩、又带 N 段新关系 → 指数增长。
- 正好打在 beachhead（海外华人女性）最痛的感情议题上。

## 2. 病毒回路（主推「邀请链接」模式）

1. **A 选关系 + 对象**：恋人/暧昧/闺蜜/家人/同事。
2. **给 A 一段勾人 teaser**（初步契合，但「要看准得…」）。
3. **闸门：让 TA 也测** → A 发邀请链接（微信/小红书/IG/WhatsApp），链接自带暖心预填话。
4. **B 收到「XX 想看你俩合不合」** → B 输入自己生辰 → **B 也走首读激活峰（B 上钩）**。
5. **双方解锁完整合盘** → B 现在也有自己的关系想合 → 回到 ①。

> **为什么主推邀请模式而非「A 替双方填」**：① A 替填零新人，邀请模式每次都可能拉新；② 把完整结果当诱饵换一个新用户；③ A 替填涉及用到第三人生辰（隐私），B 自填更干净。

**关系类型＝病毒面积放大器**：每多一种关系（尤其**家人/父母**——海外离家痛、mainland app 不碰），多一条裂变路径。

## 3. 邀请 UI/UX 铁律（简单直接）

1. **A 端**：teaser 后**一个大按钮「邀请 TA 一起测」**→ 唤起原生分享。
2. **B 端**：**一屏说清「谁邀我·为什么」+ 一个主按钮**，零解释障碍（「XX 想看你俩合不合，只差你这一半」）。
3. **先给 B 自己的首读激活峰，再要登录**——注册墙不能挡在惊艳前面。
4. **输入最少**：日期 + 地点，时间交给校准小测；标注「30 秒，跟着问就行」。

## 4. 结果设计：多维分数 + C 声音

- **多维分数讲故事**，而非单一契合度：如 ❤️‍🔥心动 95 / 🛟安全感 42 / 💬懂你 70 / ⏳长久 58。维度落差本身即洞察（「超来电但缺安全感」）。
- **顶部一个总分**当分享钩子（「我们 78%」好晒），维度条讲故事，C 声音说穿动态并兜住。
- 结果可一键「存成合盘卡」→ 回流广播（与金句卡双回路）。

## 5. 护栏（合盘比金句卡更危险）

- **分数护栏**：分数照亮、文字兜底，**永不当死刑判决**；任何低分维度必配「怎么破」；不存在全维度归零的关系；低分是「挑战」不是「分手吧」。
- **P2 镜不是预言机（加倍）**：只描述**关系的动态**（「都在等对方先伸手」），绝不下命定判决（「你们会分手」）。
- **戳准接住（加倍）**：合盘能戳到「他根本不爱你」这种要命点，戳完必须接。
