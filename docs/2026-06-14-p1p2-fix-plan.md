## 2. 新增预防规则（R10–R13，并入 DESIGN-SYSTEM §10）

- **R10 · 一次性动作必须防重放**：权限/邀请/OTP token 一旦消费即标记，重复提交→拒绝（不 overwrite）。防 [P1-2]。
- **R11 · 贵操作前先 fail-fast 校验**：API 接 user 数据，先结构完整性校验→不合格 400；只有业务异常(限流/AI 超时)才 5xx。防 [P1-4]。
- **R12 · 预期态 ≠ 错误**：public 路由的匿名访问返 `200 {authenticated:false}`，不返 401。protected 路由才 401。防 [P2-2]。
- **R13 · 不可用态必须既语义又视觉**：交互控件存在 disabled 态时必带 `disabled` attr + 视觉变化 + 测试断言。防 [P2-6]。

预防的**可执行化**（matches 现有 R7-R9 tripwire 风格）：
- 新增 tripwire `src/__guards__/chart-gating.test.ts`：扫描 gated 页，若同时出现 `router.replace(` 且读 `chart` 但未 import `useChartGuard` → 红。防 G1 复发。
- 新增共享纯校验 `isFullChart()/assertFullChart()`（chart 库），reading+chat 路由共用 + 单测。防 G2 复发。
