# Molly 设计方法 · 别为避害砍价值，沿"真vs编/自证性"划线

> **文档类型**：通用设计方法 / 自查闸（Molly 全产品适用，不限日历）
> **日期**：2026-06-16
> **缘起**：个人日历健康轨/情绪线设计中，反复出现"一刀切禁掉 → 被 Kevin 纠回精确线"的模式。把它固化成方法，下次自己走，不靠提醒。
> **配套**：[Molly 宪法 v2.1](../../2026-06-16-molly-constitution.md)（真vs编 = 本方法的判据来源）

## 0. 要根治的错误模式（先认）

碰到"有价值但也有风险"的功能，默认反应是 **禁掉 / 软化 / 求净**（"绝不X""必须净Y""把它做温柔"）。**一刀切几乎总是切多了——连价值一起砍。** 这是省事 + 怕事，不是设计。

> **黄灯触发器**：当我打算写「**禁 X / 绝不 Y / 必须净 Z / 把它软化**」——**停。** 这八成是过切。

## 1. 正确做法：五步划线（下次自己走）

1. **立约价值**：先点出这功能交付的价值，明文立约——不许在避害过程中丢掉它。
2. **锁定真害**：点出真正的害是**哪一小条**——具体的伤害**机制**，不是含糊的"可能不好"。
3. **找那根轴**：找开害与利的那条轴。**在 Molly 这产品里，它反复是同一条：真 vs 编 + 自证性**（建在真东西上、她自己答得了的 = 利；编的、她答不了的 = 害）。
4. **沿轴划，不剁**：利那一侧**全留、且拉满**（情绪强度/紧迫感/被看穿，真的就放开）；**只切编/不可自证那一侧。**
5. **危险残渣转交，不禁话题**：真正危险的部分（医疗/危机）不是封掉话题，是**转交真权威**（推去看医生）+ §9 硬底线兜住。

## 2. 判据：真 vs 编 + 自证性（同 I1）

- **真**（建在真实星象/真实事件/她真实体验上）→ 绿灯，放开、拉满。
- **编**（无中生有/夸大/冒充权威）→ 红灯，挡。
- **自证性**：她**自己答得了**的（内在态、症状、体感）→ 可问、可喂模型、可攒"了解她"；她**自己答不了**的（外部结果、医疗诊断）→ 不可断言（问了只会逼她错挂归因 = 制造恐惧）。
- 这跟全产品的 I1「验态不验果」是同一条线，只是落到不同题面（钱/健康/情绪）。

## 3. 已验案例（这套线的实证）

| 我最初一刀切 | 纠回的精确线 |
|---|---|
| 净情绪必须为正 | 情绪强度拉满，只挡**编/夸大**的负面（§8.1）|
| 绝不预言疾病 | 说**倾向**、点**可能区域**+转医生；只禁**她答不了的诊断** |
| 健康只能"不说病" | 抛身体假设让她**症状自证**→被看穿+喂模型；线=**自证性** |
| 把 B 软成 A 壳 | 直白命令保留，只把**糖（赦免）**包进去 |

## 4. 一句话

> **Molly 不做"避害式阉割"，做"沿真vs编/自证性划线"——真的、她自证得了的，放开拉满；编的、她答不了的，才挡。安全和价值是同一条线的两边，不是互相让步。**
