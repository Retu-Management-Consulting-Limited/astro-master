## 1. 「自我进化」在 Molly 里指什么（Kevin 定义）

让 **astro-master 这个产品本身**自进化——**不止内涵，而是所有维度**：UI/UX、功能、性能、文案、内涵/魂……整个产品逐轮变好、变深。

**但有边界（经因果自查 + Kevin 拍板）：是「监督式进化骑数据飞轮」，不是「全自主优化器」。**

### 1.1 总原则：一个维度能自主进化多少 = 它的「好」能被多可靠**且安全**地测量

不是"内涵特殊"——**每个维度都按同一把标尺分自主度**：能不能给它造一把「可靠 + 不可作弊」的尺子。

| 维度 | 可靠+安全的尺子？ | 自主度 |
|---|---|---|
| 性能/成本/报错 | ✅ 完全可测 | 🟢 可全自动（CI 硬闸，现在就能上） |
| 占星正确性 | ✅ 确定性校验 | 🟢 可全自动 |
| 可访问性/Web 规范 | ✅ 规则可查（已有 web-design-guidelines skill） | 🟢 可自动 lint |
| UI/UX「功能层」（漏斗/断流/卡顿） | ⚠️ 可测但可作弊（转化→暗黑模式） | 🟡 半自动 + 人/伦理闸；部分要流量 |
| UI/UX「美感层」（质感/愉悦/品牌） | ❌ 靠品味 | 🔴 变异自动，选择=人 + 用户信号 |
| 功能（该不该存在） | ⚠️ 用户信号 + 产品判断 | 🟡 半自动 + 人批准 |
| 内涵/魂 | ❌ 无可靠尺子 | 🔴 变异自动，选择=共鸣飞轮 + 品味 |

**承重修正：光"可测"不够，还要"不可作弊"**——漏斗转化可测，但自动最大化它会长出暗黑模式；参与度可测，自动最大化=上瘾设计。**自主的前提 = 可靠测量 ∧ 最大化它真的=更好（非被玩坏成伤害）。否则人留在选择位。** 这把 strategy §3 的反 Goodhart / 伦理天花板升格成全维度的通则。

> 注：UI/UX 已在手动进化（UIUX 审计已第 3 轮 + web-design-guidelines skill）。其"功能层/规范层"的部分尺子（Web 规范 lint、性能预算、可访问性、断流检测）**现在就能做成自动 CI 护栏**；美感层留给人 + 用户信号。

## 2. 机制：进化 = 变异 + 选择 + 遗传

| 进化三件套 | 在 Molly 里 | 能自动到哪 | 落在哪个 spec |
|---|---|---|---|
| **变异**（生成候选） | AI 生成：新功能、更深解读、新角度 | 越来越自动 | feedback-loop（现限文案/视觉，可往功能扩） |
| **选择**（谁活下来） | eval 尺子 + 真用户共鸣/留存信号 + **Kevin 品味** | 功能端可自动；**内涵端必须人+共鸣信号** | eval-harness + strategy 传感器 |
| **遗传**（存活固化，下一代起点更高） | 合进 main、成新基线、**喂数据飞轮** | 可自动 | CI 基线 + 飞轮 |

**自我进化 = 这个回路转起来，且每转一圈起点更高。** 自动化的是「变异」和「遗传」越来越多；**「选择压力」在内涵深端永远锚在人的品味 + 真人共鸣上**——这不是缺陷，是不退化成平庸的唯一保险。

---

## 3. 硬边界：内涵端为什么不能全自主

**内涵（深度/意义/魂）没有可靠的尺子。** 让 AI 自动优化"深度评分" → 必然 Goodhart 成**听起来很深、其实谁都适用的伪深刻废话**（巴纳姆陷阱升级到意义层）。**自动优化测不准的东西 = 退化成模仿与平庸。**

- 功能端：可测（能建、能跑、用户信号可比）→ 可较多自动 + 人批准。
- 内涵端：不可靠测 → **变异可自动，选择必须人 + 真人共鸣**。
- **决定（Kevin 2026-06-16）**：深端走「监督式 + 骑飞轮」，否决「全自主优化内涵」。
- **这是所有"不可靠测"维度的模板**：内涵、UI/UX 美感、品牌质感同理——变异可自动，选择靠人 + 真实用户信号。eval 因此不是一把尺子，是**一族按维度的尺子**；有可靠+安全尺子的维度更自主，没有的维度人当尺子。

---

## 4. Keystone：真用户共鸣数据飞轮（让内涵能"真进化"的唯一支点）

让内涵能**真进化而非模仿**的，唯一靠的是不断变厚的飞轮：

```
(星盘, 解读, 真人有没有被说中) —— 越攒越厚
        │
        ├─ 短期：回灌解读 prompt / few-shot
        ├─ 中期：当"选择压力"，筛掉假深刻、留下真打动人的
        └─ 长期：成为护城河数据资产，可检索增强/微调
```

- 有飞轮：产品**真的在学"什么深度能打动真人"并沉淀**。
- 没飞轮："自动进化内涵" = "自动生成像深刻的话" = 假进化。
- **现状：飞轮是整套里最没建的 keystone**（产品内"准度评分"入口还没有；self-model `understanding.ts` 仍是"懂你度"刻度盘，没真让解读变准）。

---

## 5. 已有的拼图怎么拼成这一个回路

| 已设计的 | 在进化回路里的角色 |
|---|---|
| feedback-loop 执行器 | **变异 + 执行**（代码层；现限文案，人工 merge 选择） |
| eval harness | **选择的传感器**（量好不好；CI 硬闸守功能端不退化） |
| strategy 八传感器 | **更多选择信号**（埋点/竞品/假人/社媒…）|
| 产品内"准度评分" | **飞轮的燃料**（一举两得：eval 金标准 + 飞轮原料）|
| 越用越准 / 校准 | **用户层的遗传**（同一个人越用越准）|
| **真用户共鸣飞轮** | **keystone：让内涵能复利**——缺 |

**一句话：自我进化不是某一个组件，是上面这些拼成一个监督式进化回路；缺的关键拼图是共鸣飞轮。**

## 6. 排序含义（这把"为什么"如何改排序）

1. **先建飞轮脊梁**：产品内"准度评分" → eval（选择尺子，CI 硬闸先守功能端）→ 校准（用户层遗传）。这条脊梁同时服务"越用越准"留存命门 + 内涵选择压力。
2. **功能端变异自动化**：feedback-loop 解冻 + 从文案扩到功能（人批准）。
3. **内涵端**：AI 大量变异深度，**选择永远 = 真人共鸣（飞轮）+ Kevin 品味**；遗传靠飞轮复利。
4. 重型传感器仍按用户量分档解冻（strategy §6）。

---

## 7. 诚实边界

- 「从功能到内涵全自动自我进化」做不到**纯自主**——内涵端无可靠测量，纯自主必退化。能做到的是**监督式 + 飞轮**的真进化。
- 内涵进化的速度受**真用户量 + 时间**约束（飞轮要真人共鸣喂）——n=10 飞轮薄，随用户长大才转得快。
- 人（Kevin）在深端是**选择压力**，不是临时帮手；这是设计的一部分，不是过渡。

## 8. 整体重审（2026-06-16）：把"质量棘轮"补成"真进化框架"

第一版框架被诚实质疑后发现：**它把"选择"做透了，却几乎没有"变异"那一侧**——是个完整的质量棘轮，半个进化框架。补上：

### 8.1 必补（框架级窟窿 + 闭法）

**① 变异引擎（不只反应用户反馈）**
原本变异只有一个来源：用户反馈（被动）。补**两条变异源**：
- **反应轨**：用户反馈/传感器洞察 → 改进（已有，feedback-loop）。
- **生成/探索轨**：AI 主动在设计空间里提候选（新功能、新解读角度、新 UI），不等用户抱怨。

**② 探索轨（Kevin 拍板：要跃迁）**
纯利用（只接受分数变好）会卡在局部最优、永不跃迁，且 eval 闸会扼杀创新。补：
- 一条**探索轨**允许**短期分数更差**的大胆实验（人拍板才上），与"利用轨"并行。
- 探索轨的产物进**预览/实验**，不进自动 CI 硬闸（硬闸只守利用轨的回归）。
- 比例可调（多少算力给探索 vs 利用）。

**③ 全局适应度 + 跨维度裁决**
per-dimension 尺子不会自动加总。定义：
- **北极星 = 留存**；各维度尺子是**约束/信号**，不是各自为政的目标。
- **跨维度冲突**（提共鸣 vs 拖慢速度）：可测维度间设权衡规则；**牵涉品味/内涵的冲突 → 人裁决**，并记录取舍（trade-off 留痕）。

**④ 延迟错配（别饿死慢但重要的）**
代码改动分钟级、共鸣周级、留存月级。快信号会主导、饿死留存/内涵（恰是首要目标）。补：
- 信号**按延迟分层**；**快指标的收益不得自动盖过慢指标的风险**。
- 留存/内涵走**单独的慢回路**（季度级评审），不与 CI 快闸抢节奏。

**⑤ 品味飞轮（让"人当选择压力"可扩展）**
只有一个 Kevin = 深端瓶颈。补一个**与共鸣飞轮并列的品味飞轮**：
- Kevin 每次 merge/否决 = 训练信号 → AI 的变异+预筛越来越能预测"Kevin 会批什么" → 预过滤、减负。
- **永不替代** Kevin 的最终选择，只**减少送到他面前的垃圾**。

### 8.2 有边界的元进化（Kevin 拍板：进软件、不碰硬机器）
- **可自进化（软）**：尺子阈值、变异/探索策略、品味模型、探索比例。
- **永不自改（硬）**：安全代码、危机检测、人工 merge 闸、本 §8 的元规则本身、`PROTECTED_PATHS`。
- 软部件的自进化**也要过人工闸**（它改的是"怎么进化"，更需监督）。

### 8.3 第二梯队（记录，实现时处理）
- **幸存者偏差**：飞轮只学留下的人 → 也要采**流失/退出信号**（不只评分者），防越优化越窄、越回音壁。
- **尺子要被审计**：授予一个维度"自主"前，先验"这尺子可靠且不可作弊"；尺子本身定期人工重校（v1 的"深"≠v10 的"深"）。
- **双优化器协调**：人类团队也在高速改产品；自动进化**必须让位于人的刻意决定**（守 PARALLEL-COLLAB），不回滚人的选择；冲突 → 人。
- **进化死胡同/回退**：产品层的"整支方向回退"机制（revert + 飞轮标注作废），防路径锁定。
- **进化成本预算**：每轮变异+评判的 token/算力预算 + ROI 阈值，防烧钱失控。

### 8.4 重审后的诚实结论
补完 8.1–8.3，框架从"质量棘轮"升为**有变异有选择有遗传、能跃迁、可扩展、有边界元进化**的真进化框架。但仍**非"完整自洽到无懈可击"**：
- 延迟错配（④）、双优化器协调（8.3）是**真难题，只能部分解**，靠运行中持续调。
- 框架越完备，越要警惕"完备的纸面"盖住"运行中才暴露的机制问题"——上线后用真实运行数据反推，比再多纸面复核有用。

## 9. 对标 Anthropic 内部优化（脚手架层能学的）

> 认知边界：基于**公开已知**的 Anthropic 做法（Constitutional AI、RLHF/RLAIF、RSP、评测文化、red-teaming、可解释性），非内部机密。

**最根本差距（结构性、绕不开）**：Anthropic 在**模型权重层**优化（RLHF/RLAIF 把"好"训进模型）；Molly 在**产品/脚手架层**优化（Claude 是黑盒 API，改不动权重）。Molly 的"模型"只能通过①脚手架 ②飞轮数据 few-shot/微调 变得更像 Molly。能碰的极限到此。

### 9.1 值得补的三样（脚手架层，已采纳）

1. **Molly 宪法**（= Anthropic 的 Constitution）→ **已起草** `docs/2026-06-16-molly-constitution.md`。明文原则，**所有变异+选择对着它量**，是适应度的价值层、eval rubric 的母本。是软部件（可进化，改需过人工闸）。
2. **打分模型毕业路径**（= reward/preference model）：现在的 LLM-as-judge 是**临时 prompt 裁判**，是起点不是终点；飞轮攒够 → **毕业成"用真人评分训练出来的打分模型"**。eval spec 的校准链路要明确指向这个终点。
3. **常态 red-team 轨**（= 系统性对抗）：一条**专设、持续**的对抗轨，攻 Molly（诱导有害/伪深刻/越界/注入/巴纳姆），不只查功能。与探索轨/利用轨并列。

### 9.2 也该补、但更轻（B 级）
- **Held-out 评测集**：留一部分盘**永不用于调 prompt**，防过拟合。
- **评测出处/可复现**：每次评测记录"哪个模型+prompt+数据集版本"出的。
- **评分者间一致性（IRR）**：哪怕 10 人，统一 rubric（用宪法）+ 测一致性，否则金标准本身是噪声。
- **灰度 + 自动回滚（canary）**：有量后，线上指标掉了自动回滚。

### 9.3 不需要 / 别 cargo-cult（明确排除）
- **机制可解释性**：Anthropic 级研究，对 Molly 过度；行为层评测够。
- **RSP / 能力阈值门**：那是防前沿模型灾难性风险；Molly 用"危机检测+人工闸"已匹配其风险。
- **海量评测规模 / 专业标注团队 / 大规模在线实验**：n=10 阶段过早；随用户量自然解锁（strategy §6）。

### 9.4 已经学到的（不是差距）
**"评测先行、没 eval 不改"这个纪律**——Anthropic 文化的灵魂——本框架已把它提为地基（eval-harness "评测先行"）。方向对。
