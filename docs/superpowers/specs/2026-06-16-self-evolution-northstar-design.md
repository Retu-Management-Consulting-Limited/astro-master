## 1. 「自我进化」在 Molly 里指什么（Kevin 定义）

让 **astro-master 这个产品本身**自进化——**不止内涵，而是所有维度**：UI/UX、功能、性能、文案、内涵/魂……整个产品逐轮变好、变深。

**但有边界（经因果自查 + Kevin 拍板）：是「监督式进化骑数据飞轮」，不是「全自主优化器」。**

### 1.1 总原则：一个维度能自主进化多少 = 它的「好」能被多可靠**且安全**地测量

不是"内涵特殊"——**每个维度都按同一把标尺分自主度**：能不能给它造一把「可靠 + 不可作弊」的尺子。

| 维度 | 可靠+安全的尺子？ | 自主度 |
|---|---|---|
| 性能/成本/报错 | ✅ 完全可测 | 🟢 可全自动（CI 硬闸，现在就能上） |
| 占星正确性 | ✅ 确定性校验 | 🟢 可全自动 |
| 可访问性/Web 规范 | ✅ 规则可查（已有 web-design-guidelines skill） | 🟢 可自动 lint |
| UI/UX「功能层」（漏斗/断流/卡顿） | ⚠️ 可测但可作弊（转化→暗黑模式） | 🟡 半自动 + 人/伦理闸；部分要流量 |
| UI/UX「美感层」（质感/愉悦/品牌） | ❌ 靠品味 | 🔴 变异自动，选择=人 + 用户信号 |
| 功能（该不该存在） | ⚠️ 用户信号 + 产品判断 | 🟡 半自动 + 人批准 |
| 内涵/魂 | ❌ 无可靠尺子 | 🔴 变异自动，选择=共鸣飞轮 + 品味 |

**承重修正：光"可测"不够，还要"不可作弊"**——漏斗转化可测，但自动最大化它会长出暗黑模式；参与度可测，自动最大化=上瘾设计。**自主的前提 = 可靠测量 ∧ 最大化它真的=更好（非被玩坏成伤害）。否则人留在选择位。** 这把 strategy §3 的反 Goodhart / 伦理天花板升格成全维度的通则。

> 注：UI/UX 已在手动进化（UIUX 审计已第 3 轮 + web-design-guidelines skill）。其"功能层/规范层"的部分尺子（Web 规范 lint、性能预算、可访问性、断流检测）**现在就能做成自动 CI 护栏**；美感层留给人 + 用户信号。

## 2. 机制：进化 = 变异 + 选择 + 遗传

| 进化三件套 | 在 Molly 里 | 能自动到哪 | 落在哪个 spec |
|---|---|---|---|
| **变异**（生成候选） | AI 生成：新功能、更深解读、新角度 | 越来越自动 | feedback-loop（现限文案/视觉，可往功能扩） |
| **选择**（谁活下来） | eval 尺子 + 真用户共鸣/留存信号 + **Kevin 品味** | 功能端可自动；**内涵端必须人+共鸣信号** | eval-harness + strategy 传感器 |
| **遗传**（存活固化，下一代起点更高） | 合进 main、成新基线、**喂数据飞轮** | 可自动 | CI 基线 + 飞轮 |

**自我进化 = 这个回路转起来，且每转一圈起点更高。** 自动化的是「变异」和「遗传」越来越多；**「选择压力」在内涵深端永远锚在人的品味 + 真人共鸣上**——这不是缺陷，是不退化成平庸的唯一保险。

---

## 3. 硬边界：内涵端为什么不能全自主

**内涵（深度/意义/魂）没有可靠的尺子。** 让 AI 自动优化"深度评分" → 必然 Goodhart 成**听起来很深、其实谁都适用的伪深刻废话**（巴纳姆陷阱升级到意义层）。**自动优化测不准的东西 = 退化成模仿与平庸。**

- 功能端：可测（能建、能跑、用户信号可比）→ 可较多自动 + 人批准。
- 内涵端：不可靠测 → **变异可自动，选择必须人 + 真人共鸣**。
- **决定（Kevin 2026-06-16）**：深端走「监督式 + 骑飞轮」，否决「全自主优化内涵」。
- **这是所有"不可靠测"维度的模板**：内涵、UI/UX 美感、品牌质感同理——变异可自动，选择靠人 + 真实用户信号。eval 因此不是一把尺子，是**一族按维度的尺子**；有可靠+安全尺子的维度更自主，没有的维度人当尺子。

---

## 4. Keystone：真用户共鸣数据飞轮（让内涵能"真进化"的唯一支点）

让内涵能**真进化而非模仿**的，唯一靠的是不断变厚的飞轮：

```
(星盘, 解读, 真人有没有被说中) —— 越攒越厚
        │
        ├─ 短期：回灌解读 prompt / few-shot
        ├─ 中期：当"选择压力"，筛掉假深刻、留下真打动人的
        └─ 长期：成为护城河数据资产，可检索增强/微调
```

- 有飞轮：产品**真的在学"什么深度能打动真人"并沉淀**。
- 没飞轮："自动进化内涵" = "自动生成像深刻的话" = 假进化。
- **现状：飞轮是整套里最没建的 keystone**（产品内"准度评分"入口还没有；self-model `understanding.ts` 仍是"懂你度"刻度盘，没真让解读变准）。

## 5. 已有的拼图怎么拼成这一个回路

| 已设计的 | 在进化回路里的角色 |
|---|---|
| feedback-loop 执行器 | **变异 + 执行**（代码层；现限文案，人工 merge 选择） |
| eval harness | **选择的传感器**（量好不好；CI 硬闸守功能端不退化） |
| strategy 八传感器 | **更多选择信号**（埋点/竞品/假人/社媒…）|
| 产品内"准度评分" | **飞轮的燃料**（一举两得：eval 金标准 + 飞轮原料）|
| 越用越准 / 校准 | **用户层的遗传**（同一个人越用越准）|
| **真用户共鸣飞轮** | **keystone：让内涵能复利**——缺 |

**一句话：自我进化不是某一个组件，是上面这些拼成一个监督式进化回路；缺的关键拼图是共鸣飞轮。**

---

## 6. 排序含义（这把"为什么"如何改排序）

1. **先建飞轮脊梁**：产品内"准度评分" → eval（选择尺子，CI 硬闸先守功能端）→ 校准（用户层遗传）。这条脊梁同时服务"越用越准"留存命门 + 内涵选择压力。
2. **功能端变异自动化**：feedback-loop 解冻 + 从文案扩到功能（人批准）。
3. **内涵端**：AI 大量变异深度，**选择永远 = 真人共鸣（飞轮）+ Kevin 品味**；遗传靠飞轮复利。
4. 重型传感器仍按用户量分档解冻（strategy §6）。

---

## 7. 诚实边界

- 「从功能到内涵全自动自我进化」做不到**纯自主**——内涵端无可靠测量，纯自主必退化。能做到的是**监督式 + 飞轮**的真进化。
- 内涵进化的速度受**真用户量 + 时间**约束（飞轮要真人共鸣喂）——n=10 飞轮薄，随用户长大才转得快。
- 人（Kevin）在深端是**选择压力**，不是临时帮手；这是设计的一部分，不是过渡。
