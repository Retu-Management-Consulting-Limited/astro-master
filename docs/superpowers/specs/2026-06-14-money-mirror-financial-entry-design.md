## 0. 一句话

> **Molly 用「钱对你到底意味着什么」当钩尖，把你的金钱故事一天天讲给你听——越讲越准、从不让你难堪。钱是天天回来的留存脊柱，爱情是炸出去的病毒尖峰；金融分发，是这根脊柱够粗以后的第二步。**

## 1. 战略判断

### 1.1 为什么用「钱」当核心钩子

钱是现代生活的**万能溶剂**——同时代理生存、地位、自由、自我价值、对未来的掌控。作为钩子比爱情有两个结构性优势：**更宽**（不分单身已婚男女老幼，人人有钱焦虑）、**更日常**（每笔消费/看价格/发薪日/「我配不配」/刷到别人过得好——环境级、永不结案的留存基质）。

### 1.2 双钩分工（定稿，且已映射到现有结构）

| 钩子 | 角色 | 落在现有哪 |
|---|---|---|
| **钱** | 🦴 留存脊柱（日常、宽、留得住） | **`财富` 主题线的升级**（本 spec） |
| **爱情/关系** | 🚀 病毒尖峰（炸点、传得开） | **已存在的 `/synastry` 合盘 + `感情` 主题**，不新建模块 |

> 关键澄清：本 spec **不新建爱情模块**。爱情尖峰复用现有合盘/感情；金钱镜是 `财富` 线的升级。两钩共存于现有 IA。

### 1.3 拒绝 naive funnel + 三条路分阶

「财运旺→去交易」机制坏：占星最易转化成「立刻做金钱动作」的人，恰是最差金融客户。成立的桥是**意义/行为桥**。

| | A · 核心钩子地基（本 spec） | B · 稳健理财/保险（**第二步**） | C · 交易/CFD |
|---|---|---|---|
| 风险 | 低 | 中 | **极高·ring-fence** |
| 何时 | 现在 | A 验证后 | 永不作 wedge |

## 2. 第一性原理与钩尖

### 2.1 六条公理（精简）

1. 没人要钱，要钱的含义 → 钩尖是「钱对你意味着什么」。
2. 受不了命不知道，也受不了命定死 → 被看穿(安心) + 钥匙在手(赋能) 必须同给。
3. 最深快感 = 被看见且不被评判。
4. **钱的主情绪按人群分裂**（羞耻只是一个寄存器：匮乏者羞耻、进取者怕掉队/争胜、自由者怕被困）。占星机关「让星盘背锅、留住尊严」对羞耻段有效，对争胜段须换成掌控/领先。
5. 人凭情绪行动、用逻辑给台阶。
6. 身份 > 行为。

### 2.2 钩尖：money-meaning 分类法（v3 钉死）

钩尖是一个**随人变形的镜片**，必须是**有限可推导**的集合，否则染色没法实现/测试。定 **6 种核心 meaning**，各有星盘签名：

| meaning | 含义 | 星盘签名（推导依据） |
|---|---|---|
| **安全** | 钱=生存底气/不慌 | 土星/月亮 落或相位 2 宫；金牛/巨蟹/摩羯 2 宫；2 宫主入 4/12 |
| **地位** | 钱=被看得起/不掉队 | 太阳/木星 在 2/10；狮子/摩羯；2-10 宫互联 |
| **自由** | 钱=不被困、能选 | 天王星/木星 在 2 宫；射手/水瓶；火星 2 宫 |
| **价值** | 钱=我配得上 | 金星主/入 2 宫；天秤/金牛 2 宫；金星强相位 |
| **掌控** | 钱=主动权/掌握命运 | 冥王/土星 在 2/8；天蝎/摩羯；8 宫强调 |
| **照顾** | 钱=护住所爱/被爱 | 月亮/金星 在 2/4；巨蟹；4-2 宫互联 |

**关键：meaning 是「主 + 次」的张力，不是单值。** 推导规则：按上表给每种 meaning 打分（2/8 宫主与占星、金/木/火/土/冥 按星座+相位），取 **top-2**：

- **两者冲突**（如 自由×安全）→ 框成**张力叙事**：「你要自由，又放不下安全感——这两股在你身上拉扯」。**冲突比单值更准、更像占星、更勾人。**
- **两者一致**（如 地位×掌控）→ 框成**叠加驱动**：「你对钱的劲，是想真正掌握自己的位置」。

meaning **星盘播种、行为精化**（§5 `behavior.ts`）：Molly 越用越懂钱对你意味着什么——「越用越准」长的是真东西。

### 2.3 勾住 = 三段力

| 力 | 作用 | 落点 |
|---|---|---|
| 被看见（钩尖刺入） | 第一次「卧槽好准」激活尖峰 | S1 揭示。**一次性、会衰减，不能当日引擎** |
| 尊严/不评判（钩不脱钩） | 敢一直待着、敢说真话 | guardrail + 见证回路。信任地板 |
| **连载希望叙事**（日引擎） | 可再生、永不结案的「我的故事在推进」 | §3 + §4。**日留存真引擎** |

### 2.4 本质 + 伦理刀

本质：Molly 是那个唯一「知道钱对你意味着什么、看见你却不评判、还把你的金钱故事一天天讲下去」的存在。**伦理 = 策略**：钱的羞耻/恐惧可卸可捅，捅短期转化高但烧毁长期信任（红线明禁）；死守治愈侧。

## 3. 日引擎：连载希望叙事 + 防罐头机制（v3 核心补强）

日留存最深的基质是**叙事**——人为「关于自己、正在展开的故事的下一页」回来。希望是主色，叙事是结构。

### 3.1 三机制

1. **连载**：有承前启后（昨天/今天/这段），不是孤立今日运势。
2. **希望主调 + 真实天气对比**：由**真实行运**塑形，有旺有慎；护住式的「慎」给对比，让「旺」是挣来的。无对比则希望成噪音。
3. **meaning 染色**：同引擎按钩尖（§2.2）上色。陈昊染「拉开身位」、秦姐染「筑更稳的墙」。钩尖是**滤镜**，不是日内容。

### 3.2 防罐头化（头号风险 §8 的机制化解药）

**确定性骨架 + 受控变化文案，分两层：**

- **`narrative.ts`（纯函数·确定性·可测）** 产出 `Chapter` 骨架：`{transitKey, tone, meaningFacet, prophecyType, angle, themeKey}`。`themeKey = transitKey + meaningFacet + prophecyType`。
- **AI 文案层（`/api/narrative`）** 把骨架渲染成散文，**输入近 14 天的 `lastChapters`**，硬指令：不得重复近期主题/开头/比喻。

**去重三招：**
1. **themeKey 轮换**：同一 themeKey N 天内不重出；若行运逼出同 transit，则**轮换 angle**（机会/警示/回顾/对比/身份 5 种镜头）或 meaningFacet（主↔次）。
2. **angle 库**：同一行运可从 5 种镜头讲，给确定性骨架足够变化预算。
3. **lastChapters 作负约束**喂 AI。

**化解「确定性 vs 保鲜」内在矛盾**：骨架确定（可测 themeKey 不重复）；文案带**按日 index 的变化种子**（变化但可复现）；保鲜由「themeKey 轮换测试 + LLM 重复度评分」双测（§6）。

### 3.3 预言：大胆预言，不报数字

唯一悬崖=可证伪的「金额×日期」（自毁准度 + 误导销售）。沿四轴：**窗口**（时机，非金额）、**命运**（去赌性，「敢转向」非「敢赌」）、**条件式**（if-then，给数字感觉、系在行为上、成了庆祝没成不追责）、**量级质感**（不给数字）。类型层：`Chapter.prophecy` 无 `{amount,date}` 字段。

## 4. 架构与单元

新金钱域 `web/src/lib/money/`：

| 单元 | 类型 | 职责 |
|---|---|---|
| `persona.ts` | 纯 (TDD) | ChartFacts → `MoneyPersona{meaning:{primary,secondary,relation:'tension'\|'reinforce'}, strengths[], blindSpot, styleTag}` |
| `narrative.ts` | 纯 (TDD) | (Persona, transits, horizon, lastChapters) → `Chapter{transitKey,tone,meaningFacet,prophecyType,angle,themeKey,hopeNote,prophecy}`；含 themeKey 去重/轮换 |
| `nudge.ts` | 纯 (TDD) | (Persona, dayTransits) → `DayNudge{tone,action}`（**收编旧财运日历的旺/慎分级算法**） |
| `guardrail.ts` | 纯 (TDD) | `validateMoneyCopy()`：六公理硬门 + 免责 + 脆弱软化 + **危机信号检测（§7.5）** |
| `safety.ts` | 纯+判 (TDD) | 日 AI 输出的**通用内容安全过滤**（自伤/危机/有害），独立于金钱红线；失败回退确定性 stub |
| `profile.ts` | 状态 | `MoneyProfile{persona, chapterLog, behaviorLog, consent, riskFlags}`；即金融接缝数据体 |
| `behavior.ts` | 纯+状态 (TDD) | 见证回应 + meaning 修正 → 精化 `meaning` + 准度；refinePersona |
| `seam.ts` | 接口（A 不实现） | `getReferralContext()` flag-off 返回 null + intent 日志 |

### 4.1 API + 成本/缓存/限流（v3 补）

- **`/api/narrative`**：input userId/chart+date → **查 KV 日缓存（key=userId+date）** → miss 则 `narrative.ts` 出骨架 + AI 渲染 + `guardrail` + `safety` + 存 KV → 返回。当天重复打开免费。
- **成本模型**（关键，区别于一次性首读的盘签名 LRU 缓存）：
  - **每人每天只生成一次**，KV 缓存当天。
  - **默认 `haiku`** 跑日叙事（便宜，约半价），`sonnet` 留给付费蓝图。
  - **惰性生成**：只对**活跃/将被 push** 的用户生成，非全量用户每天烧；非活跃用户零成本。
  - **每用户限流**（1 叙事/天）+ **全局成本监控**（STATUS 强烈建议项）。
  - 骨架（`narrative.ts`）免费确定性；只有文案渲染计费。
- **兜底**：渲染失败/超时/安全不过 → 秒出确定性 stub（Molly 既有渐进模式）。

### 4.2 留存闭环（v3 补：把 push 接进来）

> 没有 re-engagement 触发，日引擎是没发条的钟。

**生成（成本控）→ push（『你的金钱故事更新了 · 第 N 页』，贴当天 hopeNote）→ 用户回访 → 读叙事 → 见证回应 → behavior 精化 meaning → 次日更准的生成。** push 接 `TODO(push)`，**同意门控**，时点贴用户惯常打开窗（默认晨段）。**收编旧财运日历的「黄金日推送时机」。**

### 4.3 类型层硬防护

金额×日期编译期不可表达；`guardrail.ts` 是所有金钱文案唯一出口（结构化禁词：货币数字+未来日期 / 羞耻句式 / 赌性 = 硬 BLOCK）。

## 5. 变现与定价（A 验证后）

三个掏钱时刻（不靠恐惧）：被看见后想更深 / 想留住懂我的人 / 希望峰值冲动。阶梯：免费（钩尖+日叙事+基础窗口，¥0 信任地基）→ 金句卡 ¥9–19（传播）→ 金钱蓝图深读 ¥68–198（主变现，接 `TODO(key)`，用 sonnet）→ Molly 在订阅 ¥30/月、年度全家 ¥30。**变现红线：付费墙只立在「更深/更久/更趁手」前，绝不挡「安心」。** 付费件 **Slice 0 不建**。

## 6. 测试（把人性+保鲜钉成 CI）

1. **纯逻辑 TDD**：`persona.ts` 同盘→同人格（含 meaning 主次+张力判定确定性）；`narrative.ts` 结构无金额+日期、连载承前、**themeKey 在窗口内不重复（保鲜硬测）**；`nudge.ts` 行运→对的红/白/绿；`guardrail.ts`「赚50万」/「你会更穷」/「敢赌」→BLOCK；`safety.ts` 危机/有害样本→拦截+回退。
2. **guardrail = 伦理写进 CI**：六公理禁区反例语料库全接住。
3. **LLM-as-judge**：① **语气**（被看见？按 meaning 给对寄存器？有希望？给钥匙？冒数字/赌性？）；② **跨人群**：换 meaning 是否真换了寄存器（陈昊版≠秦姐版）；③ **重复度**：连续 N 天同盘叙事的相似度评分（防罐头）。
4. **E2E**：入口→钩尖揭示→meaning 修正→Day-1 开篇→日叙事→见证；**断言付费墙从不挡免费「安心」**。
5. **真人金丝雀**：林星（专业）+ 秦姐（护栏）。

## 7. 流程完整性（§9 补齐的入口/态/连接件）

### 7.1 入口/发现屏（S0）
首次怎么到 S1：`/today` 与 `/chart 财富主题` 各放一张入口卡「钱对你意味着什么？Molly 想跟你说」→ S1。

### 7.2 钩尖揭示（S1）
见 §2.2，呈现 meaning 主次张力 + 盲点(天赋暗面，甩锅星盘) + 命运预言。

### 7.3 meaning 修正（S1b）
揭示后软问「这说中了吗？」→[说中了 / 有点 / 不太像]。`不太像` → 给出**次要 meaning** 或让用户自选钱对自己意味着什么 → 喂 `behavior.ts`。这是「好准」的兜底 + 精化入口。

### 7.4 Day-1 开篇态
连载第 1 页无「承前」：「第 1 页 · 你的金钱故事，从今天开始」——引入 meaning + 第一份希望，设定连载预期。区别于第 N 页。

### 7.5 见证回路 + 危机升级 + 同意
- 见证：「我看见你在扛」非「你做到了吗」；「没忍住」零评判。
- **危机升级**：若见证/对话检测真实财务危机（债务螺旋/赌博/绝望），Molly **切换寄存器**——停财运内容，给落地安抚 + 真实资源（求助/财务咨询），绝不「幸运日」框架。detect→soften→resource。
- **同意**：存金钱行为数据前取得同意；设置页已有导出/删除（STATUS），见证 onboarding 加同意时刻。

### 7.6 状态/连接件
loading（MollyThinking）、empty（无盘→/input）、付费 teaser/墙/支付 loading/成功/失败回退/已购、push 关闭态、生成失败兜底 stub。

## 9. 范围与边界

**In scope（A）**：钩尖 money-meaning（主次张力）、连载希望叙事 + 防罐头机制、预言层、`/api/narrative` + 成本/缓存/限流、留存闭环(push)、见证回路 + 危机升级 + 同意、guardrail + safety、金融接缝（仅定义 flag-off）。

**Out of scope（YAGNI）**：真实金融产品/导流/交易（C ring-fence）；记账/绑卡；账号体系沿用 STATUS §7。

**已知项（v3 列上，不一定全做）**：
- **i18n 非翻译**：财运/搞钱框架是华人文化特定的，英文是另一套内容设计，非 next-intl 直翻。
- 旧财运日历待设计项已收编：旺/慎分级→`nudge.ts`；黄金日月度+推送→`narrative.ts`+§4.2 push。

## 10. Slice 0 —— 先验承重墙，不全量建 A

**建什么**：S0 入口 + S1 钩尖揭示（meaning 主次，星盘播种）+ S1b meaning 修正 + Day-1 开篇 + **6–8 周每日金钱叙事**（行运塑形、染色、希望主调、护住对比、连载、防罐头）。挂现有 pilot 基建。**不建**：付费墙/蓝图/全月日历/金句卡/seam。

**度量（v3 补阈值 + 先行指标 + 延窗）**：
