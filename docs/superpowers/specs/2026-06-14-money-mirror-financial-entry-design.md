# 星盘大师 · 金钱人格镜（Money Mirror）—— 成为金融产品入口的地基

> **文档类型**：战略 + 功能设计 spec（brainstorm 定稿）
> **日期**：2026-06-14
> **产品**：Molly / 星盘大师（AI 西洋占星本命盘 Web App）
> **配套**：[财运日历](../../2026-06-12-wealth-calendar.md) · [客户人物形象](../../2026-06-12-customer-personas.md) · [核心设计原则](../../2026-06-12-core-design-principles.md) · [DESIGN-SYSTEM](../../../design/DESIGN-SYSTEM.md)
> **本 spec 范围**：Path A（行为金融地基）。Path B（导流稳健理财）与 Path C（交易/CFD）只定义接缝与边界，不在本 spec 实现。

---

## 0. 一句话

把 Molly 现有零散的「财运」内容，升级成一个持续的 **金钱人格镜**：它不预言你会赚多少，而是基于你的星盘，持续照见你这个人和钱的关系——你的搞钱风格、那条会反复让你破财的行为盲点、以及今天该对它做什么。这是 Molly **赚得资格**去当金融产品入口的地基。

---

## 1. 战略判断（为什么是这条路）

### 1.1 拒绝「财运日历直接导流交易」的naive funnel

最直觉的设计是「今天财运旺 → 按钮 → 去交易/买产品」。它机制上是坏的：

> **占星最容易转化成「立刻做个金钱动作」的用户，恰恰是最差的金融客户；而真正的金融买家不会被这种方式转化。**

- 天天查「今日财运」、会在绿日真去行动的，是小漾（冲动、情绪、低余额）和秦姐（求心安）。把她们在「幸运日」推进真实金融产品 → 小额、高流失、亏损，出事就怪 Molly。这正是财运日历文档红线明禁的**掠夺**。
- 真正的金融买家陈昊（高 WTP、决策参考），凭**逻辑、深度、掌控感**转化，且高度警惕被收割。一旦 Molly 因星座推他开交易户，他立刻流失。

成立的桥梁是 Molly 自己文案里已有的**行为桥**：「你这种金星天蝎，财运的敌人是为情绪买单」。这不是算命，是**金钱自我认知**，干净地映射到合法、低风险的金钱行为（自律、储蓄、规划），而非投机。

### 1.2 三条路与分阶决策

| | A · 行为金融地基（内部变现） | B · 稳健理财/保险 导流 | C · 交易/CFD 导流 |
|---|---|---|---|
| 桥梁机制 | 「懂你的金钱行为」+ 日历升级为护身 | A 之上转介储蓄/保险/稳健 | 财运旺日 → 高杠杆交易 |
| 收入 | 中（订阅/深读，已半建好） | 中高（每转化佣金高） | 最高（即时） |
| 合规/伦理风险 | 低 | 中 | **极高** |
| 受众契合 | 全员，尤其陈昊/秦姐 | 秦姐 LTV、陈昊理性 | 错配 |
| 能否当 wedge | ✅ 地基 | ✅ A 之后叠加 | ❌ 不能 |

**Path C 的明确判断**：零售 CFD 营销若用「财运/运气」框架、瞄准情绪激起的用户、跳过适当性测试，正是触发误导销售与脆弱客户规则的东西。它**不能当入口 wedge**。若日后存在，只能是对已合格（陈昊型）用户、完整 gating 的 ring-fenced 路径，永不作漏斗。

**结论：分阶，不二选一。** ① 现在做 A（已半建好、几乎零监管风险、长出后续一切金融玩法所需的信任 + 金钱处境数据）。② 信任与数据成型后叠加 B，那才是真正的「金融产品入口」，且只指向低风险产品。③ Ring-fence C。

> 重述命题：Molly 通过**先成为一面被信任的金钱行为镜**，赚得当金融入口的资格——金融产品接在信任之后，而且是稳的那些，不是交易。

## 2. 第一性原理：钱面前的人性

设计不从功能出发，从「人碰『钱 + 命』时心里在动什么」出发。六条公理：

1. **没人要钱，人要钱的含义**——安全、被看得起、自由、被爱、「我够格」。钱是代理变量。最深的钩子不是「你会发财」，是 **「你会没事的 / 你配得上更好的」**。
2. **人受不了「命不知道」，也受不了「命定死了」**。甜点是 **「命里有这个可能（被看穿，安心）＋ 钥匙在你手上（有得选，赋能）」**，两者必须同时给。
3. **最深的快感是「被看见，且不被评判」**——被看穿是最稀缺的情感品。
4. **羞耻，是钱的主情绪**。几乎人人对钱偷偷自卑（赚太少/花错/看不懂/比不过）。加评判的产品是撒盐；**卸羞耻**的产品赢得关系。占星的神机关：**让星盘背锅，让人留住尊严**——「不是你的错，是你土星的位置」。
5. **人凭情绪行动，再用逻辑给自己台阶**。哪怕「理性」的陈昊也按「怕被淘汰」买、用 ROI 包装。先打情绪，逻辑只是遮羞布。
6. **身份 > 行为**。人不为「管好花钱」改变，会为「成为账户终于听话的那种人」改变。**给一个她想长成的身份**，别卖行为。

### 2.1 由公理推出的产品本质

> **Molly 在钱这件事上的工作，是当那个唯一「看见你、替你卸下羞耻、还信你会好起来、并把钥匙塞回你手里」的存在。** 财运是表皮，**尊严 + 希望 + 被懂**才是引擎。

一句话压到底：**人在钱面前最深的不是贪，是羞耻和孤独。Molly 不算你能赚多少，她当那个从不让你为钱觉得自己蠢的人。**

### 2.2 这如何锁死金融入口策略

凭什么一个占星 app 能当金融入口？**金融决策从情绪（恐惧/羞耻/希望）做出，而整个金融业只会说逻辑，情绪层无人服务。** Molly 独占钱的情绪层。她成为入口，是因为她是这段关系里**唯一在「钱的决策真正发生的那个情绪高度」上陪着用户的人**。你不把理财产品推荐给「信你预测」的人，你推荐给「觉得你是唯一没让他为钱自卑过」的人。

### 2.3 一把必须守住的刀（伦理 = 策略）

公理 4 是双刃：同一洞察能治愈也能收割。「钱的羞耻」可卸（「不丢人」）也可捅（「你这样会更穷，快买这个」），后者短期转化更高。但捅羞耻 = 红线明禁的掠夺，且烧掉策略赖以成立的长期信任。**本设计死守「治愈」侧**——这刚好也是唯一能长出可变现信任的那一侧。短期收割与长期入口，二选一，本设计选后者。

## 3. 预言层：大胆预言，不报数字

人性要预言（公理 1、2），但有一道**唯一的悬崖**：**可证伪的「金额 × 日期」**（「你明年赚 50 万」）。两条机制使它对本策略致命：

1. **自毁要攒的信任**——准度感知是留存命门（STATUS §8.1）。数字配日期会被现实核对、常错、被用户抓到。你无法同时是「错许我发财的 app」和「我信它推荐金融产品的 app」。
2. **把 A 变成 C**——预言收益 +（日后）产品推荐 = 误导销售/责任。C 的毒漏进地基。

**合成方案：预言体验，不预言数字。** 沿四条 trust-safe 轴大胆预言：

- **窗口预言（时机）**：「今年 9–11 月是你三年来最强的搞钱扩张窗口」——方向性、有时间、真实（木星行运确是扩张窗口），是机会不是保证金额。
- **命运预言（人格）**：「你这辈子的钱，不靠死工资，靠一次**敢转向**」——像预言、深satisfying、结构上不可证伪。（注：去赌性——不用「敢赌」，赌是羞耻螺旋。）
- **条件式预言（if-then，最强）**：「这半年你要是守住情绪化消费，年底大概率会看到一个你没见过的数字」——给数字的**感觉**，把结果系在用户自己的行为上；「if 你守住」这句**本身就是行为桥**。（注：成了就庆祝，**没成不追责**——删掉甩锅暗面，否则制造愧疚。）
- **量级质感，非数字**：「不是小数目，是够你喘口气的一笔」——给质感不给数字。

**定位措辞**：从「镜不预言」改为 **「预言体验、不预言可证伪的金额」**。这条在 §4 落成类型层 + guardrail 的硬约束，不靠自觉。

## 4. 架构与单元

套 Molly 现有模式（纯 `lib/` 确定性计算 + AI 写文案围绕注入事实 + `/api` 路由 + KV/local 持久）。新金钱域置于 `web/src/lib/money/`。

| 单元 | 类型 | 职责 | 依赖 / 接现有 |
|---|---|---|---|
| `persona.ts` | 纯函数 (TDD) | ChartFacts → `MoneyPersona{strengths[], blindSpot, styleTag}` | 复用 `lib/astro` 星位事实（金/木/火/土 + 2/8 宫） |
| `prophecy.ts` | 纯函数 (TDD) | (Persona, transits, horizon) → `ProphecySet{windows[], destinyLine, conditional[], magnitudeTexture}` | 行运用现有 transit 计算 |
| `nudge.ts` | 纯函数 (TDD) | (Persona, dayTransits) → `DayNudge{tone, action, tiedToBlindSpot}` | **升级 `lib/astro/wealth.ts`** |
| `guardrail.ts` | 纯函数 (TDD) | `validateMoneyCopy()`：红线硬门 + 免责 + 脆弱用户软化 | 红线 + 六公理禁区集中地 |
| `profile.ts` | 状态 | 持久 `MoneyProfile{persona, behaviorLog, consent, riskFlags}` | 现有 KV/local；**即金融接缝数据体** |
| `behavior.ts` | 纯+状态 (TDD) | 记录「接住没」→ 算准度感知 → refinePersona | **激活现为 stub 的 self-model（§8.1 命门）** |
| `seam.ts` | 接口（A 不实现） | `getReferralContext()` 默认 flag-off 返回 null + intent 日志 | 定义 B 契约，零外部调用 |

### 4.1 两个落在类型层的硬防护

- **金额 × 日期 = 编译期不可表达**：`ProphecySet` 里**根本没有** `{amount:number, date}` 这类字段，prophecy 只能产出 window/destiny/conditional/texture 结构。"不报数字"靠类型系统，不靠提醒模型。
- **`guardrail.ts` 是所有金钱文案的唯一出口**：`/api/reading`、`/wealth` 渲染、deep read 全过它。做结构化禁词扫描（货币数字+未来日期 / 羞耻句式 / 赌性 = 硬 BLOCK）+ 强制挂轻免责 + 脆弱信号软化。

### 4.2 付费深读

走现有 `/api/reading` 加 `moneyBlueprint` 模式（接 `TODO(key)` 付费门）。AI 写文案，persona + prophecy 注入确定性事实（沿用「模型不许编星位/金额」铁律）。

### 4.3 第一性原理对每块的重写（语气基线）

- **金钱人格** → 不是「优势+盲点」，是「你和钱的关系真相，说出来、把锅甩给星盘、把尊严留给你」。盲点 = **天赋的暗面**（「你为情绪买单，是因为你比别人更敢爱、更敢活」）+「这不是你的错」。卸羞耻，不是夸。
- **日历** → 不是教练，是**每日希望配给 + 偶尔一道护身符**。希望打头；自律永远「护住式」（「我拦你，是怕你被自己坑」），不是禁令。
- **预言** → 见 §3。删赌性。
- **回路** → 不是审问，是**见证**：「我看见你这阵子在忍、在扛」，不是「你做到了吗」。
- **保险丝** → 守的不是合规，是这段关系：不撒数字谎、不制造羞耻。

> 身份从「教练」彻底换成 Molly 本来的「懂你、护住你的人」。「教练」带进管教/费劲/被盯的错误情绪。

## 5. 变现与定价（不捅羞耻地赚钱）

### 5.1 三个掏钱时刻（都不靠恐惧）

- **被看见后想看得更深**（好奇/身份）——尊严正向，林星/陈昊甜区。
- **想把懂我的人留在身边**（关系延续）——秦姐/小漾，订阅。
- **希望峰值的冲动**（非恐惧峰值）——金句卡，小漾在 hope 高点晒。

### 5.2 变现阶梯（套现有屏 + 人物 WTP）

| 层 | 内容 | 触发器 | 定价 |
|---|---|---|---|
| **免费**（钩子） | 金钱人格揭示 + 每日希望配给 + 基础预言窗口 | 第一句「好准」 | ¥0，信任地基，绝不收费 |
| **金句卡**（传播） | 「我的搞钱命格」可晒卡 | hope 峰值冲动 | ¥9–19 单次（小漾，喂 K 因子，套 `/share`） |
| **金钱蓝图深读**（主变现） | 陈昊级完整报告，确定性星位注入 | 想更深懂自己 | ¥68–198 单次（陈昊/林星，接 `TODO(key)`） |
| **Molly 在·订阅**（留存） | 持续陪伴 + 每日预言 + 推送 + 见证回路 | 留住懂我的人 | ¥30/月；¥30 年度全家（秦姐 LTV） |

### 5.3 变现红线（定价层的保险丝，第一性原理导出）

> **付费墙只能立在「更深 / 更久 / 更趁手」前面，绝不立在「安心」前面。**

可对深度/陪伴/工具收费；**永不把「知道你会不会没事」扣作人质**——那是羞耻勒索。免费层必须永远能让用户**被看见、且觉得自己会没事**地离开。本条在 §6 做成可测硬门。

### 5.4 延伸到 B

日后导流稳健理财的佣金，同一条红线照管：话术是「你现在这个阶段，配得上一个更稳的位置」（尊严/希望），不是「你这样下去会穷死」（羞耻）。`seam.ts` 现在就存好「同意 + 处境」，B 接入时不必再捅一次。

## 6. 测试（把人性钉成可执行的门）

沿用 Molly 现有：纯逻辑 TDD + Playwright E2E + LLM-as-judge。关键是把第一性原理软规则钉成硬测试。

1. **纯逻辑 TDD**：
   - `persona.ts`：同盘 → 同人格（确定性）。
   - `prophecy.ts`：断言输出结构里**不存在**「金额 + 未来日期」字段（"不报数字"的硬证据）。
   - `nudge.ts`：人格 + 当日行运 → 正确红/白/绿 + 绑对盲点。
   - `guardrail.ts`：「你明年赚 50 万」→ BLOCK；羞耻句「你这样会更穷」→ BLOCK/软化；赌性「敢赌一把」→ BLOCK。
2. **guardrail = 伦理写进 CI**：六公理禁区（数字×日期 / 羞耻句式 / 赌性 / 付费墙挡安心）做成反例语料库，guardrail 必须全部接住。
3. **LLM-as-judge 评语气**（改造 `agent-test-runner`）：样本盘生成金钱文案 → **六公理 rubric** 打分：被看见？卸羞耻还是撒盐？有希望？给钥匙（agency）？冒出数字/赌性？
4. **E2E（Playwright）**：人格揭示 → 日历 → 预言 → 付费墙 → 见证回路全链路；**关键断言：付费墙从不挡免费层的「安心」内容**。
5. **真人金丝雀**（不可自动化）：**林星**（专业及格线：星位真不真、够不够深）+ **秦姐**（护栏生死线：被看见还是被评判/被收割）。

**最承重度量**：信任/准度感知曲线 + 语气有没有滑向羞耻。guardrail eval（机器）+ 金丝雀小组（真人）是两道闸。

## 7. 范围与边界

**In scope（A）**：金钱人格、财运日历升级、预言层、付费深读、见证回路、guardrail、金融接缝（仅定义契约 + flag-off）。

**Out of scope（YAGNI）**：不接任何真实金融产品、不导流、不碰交易/CFD（C ring-fenced）；不做记账/绑卡（更重的 B+ 决策，验证后再说）；账号体系/真实后端沿用 STATUS §7 既有进度，本 spec 不重做。

**接缝**：`seam.ts` 定义 `ReferralContext`（MoneyProfile + 同意 + 风险标记），A 阶段 flag-off 返回 null、仅记录 intent 信号，B 实现时从此口接出，不回头重改。

## 8. 风险与缓解

| 风险 | 缓解 |
|---|---|
| 语气滑向「教练/说教」→ 反留存 | 身份锁死「懂你的人」；日历希望打头；guardrail + LLM-judge 评语气 |
| 见证回路逼用户认错 → 流失/撒谎 | 零评判、失败也接住；优先间接推断，不审问 |
| 预言制造愧疚/赌徒 | 条件式预言不追责；命运预言去赌性；类型层禁数字×日期 |
| 过度免责戳破权威/魔力 | 免责轻量、边缘化（页脚/护住式），不在脸上 |
| 变现捅羞耻（短期诱惑） | 付费墙红线：绝不挡「安心」；做成 E2E 硬断言 |
| self-model 仍是 stub → 「越用越准」落空 | `behavior.ts` 把见证数据真正回灌人格精化 |
