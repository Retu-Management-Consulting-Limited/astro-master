## 2. ⚠️ 一个真冲突，必须先解

`content-freshness.test.ts:70` 现在断言「本月决定性日（非 ping）**≥9/30**」——这是**红绿的下限**；而 charter **I2b 要"平淡为底、红绿稀有"**——是**上限**方向。两者顶。

**解法（不是二选一）**：把"按天真变"和"红绿稀有"解耦——
- **新鲜度** = 今日格**内容**邻日必变（连平淡日也靠月亮态/备战糖变）→ 强断言留着、但断言的是**呈现层文案**，不是"必须有 N 个红绿日"。
- **稀缺** = 红绿**状态**月度有上限。
- 即：`freshness ≠ chargedness`。改 guard：断言"邻日今日格文案 not.toBe"（包括平淡日），删掉/改写"≥9 决定性日"那条下限。

**P1 · 核心循环对齐（de-risk 切片的脸）**
