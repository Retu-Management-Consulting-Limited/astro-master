## 1. 问题（数据诊断，非猜测）

192 盘 × 2026 实测当前日信号：

| 指标 | 现状 | 判定 |
|---|---|---|
| 跨盘相似度（6月两两相关） | 0.08 | ✅ 已个性化 |
| 月度自相似（6月vs7月） | −0.22 | ✅ 不重复 |
| **决断度** | **平 69% / 旺 16% / 慎 15%** | ❌ **七成是「平」=温吞无感** |
| 日变化来源 | 月亮快层占 68% | 纹理靠月亮、常擦边压不过阈值 |

**根因不是个性化或重复（都好），是决断度**：分数常年贴 50±10，很少破 64/42，七成日子在说「财运平稳，按计划走」。且现模型**不说「为什么」**——只有泛泛的旺/平/慎。

**「切身体会」的真实抓手 = 决断度（少点平）+ 具名驱动（说清因果）。** 现模型慢层只用木/金/土、快层只用月亮，没用上真实的**金星/水星/火星/太阳行运打到你财库**——既更决断、又天然带名字。

## 2. 设计总览（双层）

### Layer 1 — 每日纹理（变决断）· 改核心分数

```
日分 = 50 + moonFast(不变) + eventPressure
```

`eventPressure(date)` = 把现「慢层」从「木/金/土」**扩成 5 行星**对金钱点的真实行运，带增益与封顶（**水星不进日分**，仅作 Layer 2 事件——见标定结论）：

```
raw = Σ_{mp ∈ moneyPoints} [
   +8·harm(sep(tJup,mp),[0,60,120],10)     // 木星 吉
   +6·harm(sep(tVen,mp),[0,60,120], 6)     // 金星 吉
   +4·harm(sep(tSun,mp),[0,120],     5)     // 太阳 吉(弱)
   −8·harm(sep(tMar,mp),[0,90,180], 6)     // 火星 凶
   −8·harm(sep(tSat,mp),[0,90,180], 9)     // 土星 凶
]
eventPressure = clamp(g · raw, −CAP, +CAP)
```

- `moneyPoints` = 现有 4 点（金星、木星、2宫主、8宫主）。**不加福点/11宫**（沿用否决案结论）。
- **`g = 1.5`、`CAP = 36`（已 harness 锁定）**：平% = 52%（落在验收门 [48,58]）；CAP 仅 1.1% 的日子触顶（安全不常 binding）。
- 判级阈值（旺≥64 / 慎≤42）**不变**。
- 本层**取代**现 `slowWealth`（保留其金钱点 + 土星对称凶层的成果，只是扩行星集 + 加增益）。

**实测（192 盘, g=1.5, CAP=36, 水星 off）**：平 69%→**52%**，旺/慎 **24/25 对称**（|旺−慎|=1，凶星平衡吉星 → 无绿洪）。

> **标定结论（原 §8.3 待决）**：水星**计入日分**会把旺推到 28、|旺−慎|=6（吉性偏斜）；**剔除**后旺/慎=24/25、|旺−慎|=1 且平% 仍 52。故 **水星不进日分，只保留 Layer 2「水星谈钱」事件**。

### Layer 2 — 具名大事件（少而精）· 新内容面

检测**慢一点的金钱行星**（金/木/火/土，可含水星合相）对金钱点的**紧密相位**，聚成**窗口**、具名带吉凶：

| 行星·相位 | 名称 | 吉凶 | 行动文案方向 |
|---|---|---|---|
| 金星 谐相 | 金星照财库 | 🟢 | 开口要钱/谈合作/收款 |
| 木星 谐相 | 木星扩张财运 | 🟢 | 推进、谈大单 |
| 火星 硬相 | 火星冲财·易冲动破财 | 🔴 | 缓大额、别冲动下单 |
| 土星 硬相 | 土星压财·紧手谨慎 | 🔴 | 收紧、别加杠杆 |
| 水星 合相 | 水星谈钱·宜签约谈判 | ⚪ | 签约、对账、谈判 |

- **事件 = 窗口**：把 (行星, 相位类) 的连续命中日（间隔≤2 天）聚成一个窗口 `{planet, name, valence, startDay, endDay, peakDay}`。
- **频率门**：调相位集/orb 使 **2–4 窗口/月/盘**（实测：仅合相 orb≤3 太稀=0.6/月；全相位太密=68% 日）。起点：**合相 orb≤5；冲/刑/拱 orb≤3；水星仅合相 orb≤4**；实现期 harness 锁定落入 [2,4]。
- **当日主驱动** = 当天 eventPressure 5 项（木/金/日/火/土）里**绝对值最大**且 |项| > 1.5 的那一项 → 写进当日文案（"今天金星照你财库——…"），无显著项则回退泛文案。（水星不在日分项里，故不作主驱动；它只出现在事件窗口。）
- **事件 orb（已锁定，2.3 窗口/月）**：金星/木星 `[0:orb5, 60:3, 120:3]`；火星/土星 `[0:5, 90:3, 180:3]`；水星 `[0:4]`（仅合相）。

**Kevin 6 月实测故事**：6/3-8 金星照财库（75-90 分）→ 6/9-16 火星冲财（6/10 压到 46）。具名、有因果、可行动。

## 3. 数据结构

```ts
type Valence = 1 | -1 | 0;
interface Driver { planet: BodyName; name: string; valence: Valence; }      // 当日主驱动
interface EventWindow { planet: BodyName; name: string; valence: Valence;
                        startDay: number; endDay: number; peakDay: number; }

interface DayWealth { day; level; intensity; retro: BodyName[];             // 现有
                      driver?: Driver; }                                    // 新增
interface MonthWealth { days: DayWealth[]; goldenDays: number[];           // 现有
                        events: EventWindow[]; }                            // 新增
```

`driver`/`events` 纯加法，不破坏现有字段；`level/intensity` 仍由分数决定。

## 4. UI（`web/src/app/wealth/page.tsx`）

1. **日历格**：事件窗口日加小标记（行星字形或吉凶色点），peak 日加重。
2. **当日详情**：有 `driver` 时用**具名文案**（「今天**金星照你财库**——开口要钱、谈合作赢面最大」），无则回退现泛文案。保留水逆条。
3. **「本月大事件」迷你列表**（日历上/下方）：2–4 条具名窗口（`6/3-8 金星照财库 🟢`）——强规划价值、可晒。

## 5. 文案红线（沿用 [财运日历设计 §3-4]）

- **行动时机非预言**：说「适合/避免做什么」「赢面偏高」，**不**说「你会赚 X / 财运 90 分」。
- **不怂恿赌博/投机**；不卖招财改运。
- 吉凶→动词：🟢 要钱/谈/收款；🔴 缓大额/别冲动/别加杠杆；⚪ 签约/对账/谈判。
- 页脚轻免责保留。

## 6. 测试（TDD + freshness）

**单元（TDD 锚点，真实星历）**：
- `eventPressure` 有界 [−CAP, CAP]；`g=0` ⇒ 退化为「无事件层」基线（零回归对照）。
- **决断度验收**：192 盘集 平% ∈ [48,58]、|旺%−慎%| ≤ 6（防绿洪）。
- **事件聚合**：连续命中日（间隔≤2）合成一个窗口；间隔>2 断开。
- **主驱动** = 当日绝对值最大项（构造一天 Venus 主导 vs Mars 主导，断言 driver.planet）。
- **具名窗口确定性锚点**：Kevin 2026-06，金星窗口含 6/3-8、火星窗口含 6/9-16（`toContain`）。
- **频率门**：合成盘集事件窗口 2–4/月（均值断言）。

**freshness 强断言**（登记 `web/src/__guards__/content-freshness.test.ts`）：
- 相邻日 `driver?.name` / 分数**可变**（不是冻结）。
- **不同盘 → 不同 events**（最强形式 `not.toBe`，禁 `Set.size>1`）。

**e2e**：事件 banner + 具名文案在已知事件日渲染（用点选已知事件日，clock-free）。

## 7. 护栏与上线

- **防绿洪**：凶星对称（已验旺/慎 23/21）+ 验收门 |旺−慎|≤6。
- **防噪声**：事件 orb 收紧 + 窗口聚合 + 频率门 2–4/月。
- **不破坏既有**：`driver/events` 纯加；retro/goldenDays 不动。
- **流程**：单 PR（L1+L2）→ tsc·unit·build·e2e 四闸 → 浏览器呈现层验证 → 部署前打印 192 盘「现状 vs 新」分布确认验收门 → `vercel --prod`（部署前显式确认）。

## 8. 标定旋钮（已用 192 盘 harness 锁定）

| 旋钮 | 锁定值 | 实测达标 |
|---|---|---|
| `g`（日分增益）/ `CAP` | **1.5 / 36** | 平%=52 ∈[48,58]；CAP 仅 1.1% 触顶 |
| 事件相位集/orb | 金木 `[0:5,60:3,120:3]`、火土 `[0:5,90:3,180:3]`、水 `[0:4]` | **2.3 窗口/月** ∈[2,4] |
| 水星是否进日分 | **否**（仅作事件） | on→\|旺−慎\|=6 偏斜；off→=1 平衡 |

> 三个旋钮均已达标锁定，实现按本表硬编；部署前再跑一次 192 盘分布确认未漂移。

---

## 9. 一句话

现模型已个性化、不重复，但**七成日子温吞、且不说为什么**。本设计用真实的金/木/火/土/日/水行运把决断度从「平 69%」拉到「~52%」，并把它们聚成**具名大事件**（金星照财库 / 火星冲财）——决断 + 具名因果 = 切身体会。凶星对称防绿洪、orb+窗口防噪声、harness 先标定后上线。
