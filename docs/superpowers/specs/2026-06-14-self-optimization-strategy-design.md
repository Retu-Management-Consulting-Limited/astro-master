## 1. 目标与锚定

让 Molly 成为**会自我优化的平台**：测试用户反馈只是一个传感器，目标是一整套传感器持续喂养自动优化。

- **首要指标（Kevin）：留存 / 复访。**
- **留存的主杠杆 =「越用越准」的共鸣感**，且它本身就是个留存回路（越用 → 越准 → 更想用）。
- ⚠️ **承重风险**：「越用越准」必须是**真的**——若 self-model（现 stub）没让解读随使用变准，这句是空头支票，反而是留存杀手。校准回路的第一要务是把这句话变真。

### ⚠️ 机制诚实（不可绕过）
**n≈10 内测、未放大、时间窗口未走完 → 现在测不了留存**（没有 cohort 算复访曲线）。所以 Phase 1 的「留存」工作不是「测→改→再测」，而是：① 建**度量地基**（eval harness）；② **拉留存杠杆**（把「越用越准」做真）；③ **质性地问那 10 个人会不会回来、为什么**。测量驱动的留存迭代，等有 cohort 才解锁。

## 2. 架构

```
   传感器（产出「带类型的洞察」）              执行器
 ┌──────────────────────────┐
 │ 0 自评 eval harness（地基）│         ┌─ 类型路由 ─────────────────┐
 │ 1 行为埋点挖掘            │         │ eval 失败 → 回归修复        │
 │ 2 准度/共鸣校准回路 🔥    │  洞察    │ 文案/视觉小修 → 自动 PR     │
 │ 3 AI 假人测试            │ ──────▶ │ 竞品/质性洞察 → 人工 roadmap│
 │ 4 竞品情报               │  (typed) │ 漏斗数字 → 实验假设         │
 │ 5 评论/社媒聆听          │         └────────────────────────────┘
 │ 6 技术/成本信号          │           执行器 = 已设计的 feedback-loop 双轨
 │ 7 变现/增长信号          │           （小修自动 PR / 其余分拣邮件，人工 merge 闸门）
 └──────────────────────────┘
```

**两条修正（来自专业碰撞）：**
1. **eval harness 是第 0 位地基**，不是 Phase 1 的防守配菜——评测先行（eval-driven development）：没有「质量怎么量」的尺子，任何校准改动都不知是改好还是改坏，也挡不住 prompt/模型一动就回归。
2. **洞察队列是「带类型路由」，不是一根管**——不同传感器产出不同类型洞察，去向不同；只有「文案/视觉小修」才进那个自动改 PR 的执行器。

## 3. 两个被拆开的「准度」（关键概念修正）

| | 占星正确性 | 共鸣感 / 被看穿感 |
|---|---|---|
| 是什么 | 解读对不对得上真实星盘 | 用户*觉得*「这是专门写给我的、说中了」 |
| 现状 | 已守（模型不许编星位） | 未度量 |
| 谁驱动留存/口碑 | 间接 | **直接** |
| 怎么量 | 确定性校验 | 人/LLM 判「像不像专门写给我」 |

**校准回路与 evals 都瞄「共鸣感」，不是占星正确性。**

### ⚠️ 反 Goodhart / 巴纳姆护栏（必须，否则有毒）
自动优化「共鸣感评分」会把模型训成写**放之四海皆准的冷读套话**（谁都觉得准）——分高但空洞，对情绪脆弱用户不道德。
- **反套话检验**：每段解读过一道「这段只适用这张盘 vs 谁都适用」判定；越通用越扣分。
- **伦理天花板**：优化目标是「真被看穿」不是「最大化回访/依赖」；情绪脆弱人群的留存优化设上限。
- **人类品味守门人**：共鸣感的最终裁判是人（Kevin/指定的人），不纯交给指标。

## 4. 校准回路 = 数据飞轮（不止实时改文案）

校准回路的产物不只是「改一句文案」，而是攒一个**专有数据集** `(星盘, 解读, 用户共鸣信号)`：
- 短期：信号回灌解读 prompt / few-shot 例子。
- 长期：成为 Molly 的护城河数据资产，可做检索增强 / 微调。
- **把校准回路当「在建数据飞轮」设计，而非一次性反馈通道。**

## 5. 八传感器全景（菜单，留存锚定）

| # | 传感器 | 留存杠杆 | 现在能跑 | 工作量 | 最大坑（诚实） |
|---|---|---|---|---|---|
| 0 | **自评 eval harness** 🧱 | 度量地基，守命门质量 | ✅ n=0 也回本 | 中 | 判官也是 AI，需固定基准集 + 人校准 |
| 2 | **准度/共鸣校准回路** 🔥 | 直击命门 + 数据飞轮 | ✅ 10 人有信号 | 中高 | self-model 现 stub；防巴纳姆 |
| 4 | **竞品留存机制情报** | 偷每日回访机制 | ✅ 现在 | 低 | 聚焦留存机制，别变抄功能 |
| 6 | **技术/成本信号** | 慢/报错=流失 | ✅ 现在（cost.ts 已有） | 低 | 接 Sentry（TODO obs）即得 |
| 1 | **行为埋点挖掘** | 哪些功能/路径复访高 | ✅ 机器现建，数据等 cohort | 中 | 10 人样本薄，先建机不先下结论 |
| 3 | **AI 假人测试** | 抓断流/卡点/跑调 | ✅ 现在 | 中高 | **对「会不会回来」几乎无效**——假人没真情感拉力，强在找 bug/摩擦 |
| 5 | **评论/社媒聆听** | 人群为何留/走 | 竞品评论 ✅现在 / 自有 🔒等规模 | 中 | 小红书抓取有 ToS/反爬 |
| 7 | **变现/增长信号** | （非留存） | 🔒 付费墙已暂缓 | — | 现在抬不动，后置 |

> 补：**质性用户访谈（可 AI 主持）** 是 n=10 时最便宜也最诚实的传感器——直接问那 10 个人「会不会回来、为什么」，胜过薄埋点。列入 Phase 1。

## 6. 分期 Roadmap（已瘦身 — Kevin 选「Phase 1 只建地基」）

### Phase 1 · 现在（n≈10）——只建地基，不烧空转
1. 🧱 **自评 eval harness**（共鸣感 + 占星正确 + 安全 + 反巴纳姆的基准集与评分；n=0 即回本，护住 prompt/模型改动不回归）
2. 🗣️ **质性用户访谈**（AI 主持，问那 10 个人留存意愿与原因）
3. 👁️ **人工/AI 读那 10 个人的真实使用**（Kevin 或一个人亲自看 + AI 辅助归纳）
> 重型传感器（埋点挖掘 / 竞品情报自动化 / 社媒 / 假人）**按用户量分档解冻**，不在 n=10 建。

### 用户量解冻闸（heavy sensors 何时上）
| 闸 | 解冻 |
|---|---|
| 有稳定日活 cohort（数十+） | 行为埋点挖掘（留存 cohort 分析真正跑） + 竞品情报自动化 |
| 自有评论/社媒有量 | 评论/社媒聆听 |
| 有足够流量做对照 | A/B 实验框架 |
| 付费墙上线 | 变现/增长信号 |

### 全程
所有传感器把**带类型洞察**丢进队列 → 类型路由 → 复用双轨执行器（feedback-loop，需要时解冻）。
