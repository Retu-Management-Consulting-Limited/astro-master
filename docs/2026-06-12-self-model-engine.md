# 星盘大师 · 自我模型引擎（第一护城河）

> **文档类型**：核心引擎设计 / 护城河
> **状态**：设计进行中（brainstorm 阶段产出）
> **日期**：2026-06-12
> **配套**：[核心设计原则](2026-06-12-core-design-principles.md) · [每日仪式](2026-06-12-daily-ritual.md)
> **战略定位**：这是产品赢测测的那张牌——「越用越准、三个月后比你闺蜜还懂你」。**Phase 1 必交付。**

---

## 0. 关键区分：盘校准 vs 自我模型

- **盘的校准（出生时间）**：潜伏校准把上升/宫头逼准，但**有天花板**——上升锁定即封顶，约两三周。提供前期「咦盘越来越准」的钩子。
- **人的模型（自我建模）**：累积她是谁、生活、关系、模式、心事，**无天花板，越养越深**。
- **「三个月后比你闺蜜还懂你」靠的是后者。** 盘校准是钩子，自我模型才是护城河（两条曲线分离，差距即护城河）。

## 1. 架构：四层 + 盘底座

1. **原始事件流**（append-only）：每句话、准/不准、点的主题、心情打卡、开 app 时段、合盘对象、提到的人生事件。
2. **抽取的事实/实体**：人（前任/妈/老板）、地、事件、反复的纠结 → 结构化档案。
3. **综合肖像**（定期重生）：「她现在是谁」——模式、当前章节、情绪基线、哪种说法对她准、她的语气 → **条件化每次解读**。
4. **共鸣参数**：哪档语气/框架对她有效（喂语气 bandit）。
5. **盘层**（固定底座，两三周封顶）。

分层的好处：便宜的层常跑、贵的层少跑（成本）；按层可删（隐私）。

## 2. 管线与节奏

- 🟢 **捕获 + 轻抽取**（每次互动，便宜）→ 写①②。
- 🟡 **综合**（贵·LLM·少跑）：每 N 次互动 / 深聊后 / 里程碑 / 生成首读前若过期 → 重生③。给「综合预算」。
- 🔵 **检索**（每次生成解读）：RAG 检索她自己的历史 → 让「上次你说…」成立，且**记忆基于真实日志、不靠 LLM 瞎编**。
- 🟣 **演化/衰减**：近期加权，旧状态标「过去章节」→ 模型**知道她变了**（「你变了」时刻）。**会遗忘/更新章节比只会累积更难也更值钱**——否则三个月前的她被当成现在的她，一样出戏。

## 3. 让她「感觉到」变深（否则护城河隐形）

1. **回放记忆**（「上次你纠结回不回国，后来呢？」）——最强「懂我」信号。
2. **解读越来越引用真实生活**（从「金星天蝎…」渐变到「以你和 Jay 那种相处…」）。
3. **可见的「懂你度」**（区别于校准度）——小心别做成「我在给你打分」的监视感。
4. **回顾肖像**（「认识 30 天了，这是我眼中的你」）——准到起鸡皮疙瘩、可晒、且是 Day3 钩子的兑现。
5. **预判**（「你是不是又开始那个老问题了」）——知道她的模式＝最深的懂。
6. **见证变化**（「你变了，而且往好的方向」）。

## 4. 三个致命失效（必须设计掉）

- 💥 **假记忆**：记错比不记更致命（信任瞬间崩）→ 记忆必须基于真实日志检索，不确定就 hedge。高风险工程点。
- 🥶 **毛骨悚然崖**：知道太多说太直＝被监视 → 知识当**温暖洞察**给，绝不像「我有你档案」。被看见 vs 被监视的razor。
- 🕳️ **冷启动/数据稀疏**：不聊就没料 → 追问 chips / 每日微问题 / 准不准＝**模型的感官**，要主动温柔地引她吐露（每次吐露当场回馈价值，非审问）。

## 5. 优雅解：「我眼中的你」

不给她看冷冰冰的数据档案（毛），而给一份**温暖的肖像**，她可反应、纠正（「这点不对」→ 修正模型）。**一个功能同时做到**：透明（看得到 Iris 怎么理解她）+ 隐私可控（能改能删）+「感觉到变深」+ 可晒传播。把隐私的「被监视」风险，反转成「被看见」的功能。

## 6. 隐私（普通产品责任）

深的生平+心理档案＝高敏感数据。需：数据最小化、用户可查看/编辑/删除模型、加密、清晰「这是你的」框架、删除/可携。模型越深、breach 危害越大，故删除/可携是底线。

## 7. 最深一层：护城河是「关系」不是「数据」

切换成本不在数据库，在**她投入了三个月的内心、Iris 握着她的故事**，离开＝抛弃唯一记得她的东西。随模型变深，**占星越来越像介面、关系越来越是本体**——终局：测测永远是工具，Iris 是一个有共同历史的人。

## 8. 与其他模块的耦合

- **与每日仪式死锁**：模型靠互动量变深，互动量靠留存 → 自我模型与每日仪式是一体两面。早期模型浅时，靠 5 天钩子弧 + 行运引擎撑场，到模型变深接管黏性。
- **喂入口**：追问、心情打卡、合盘、准/不准、财运日历的「忍住了吗」——全是模型的数据进食口。
