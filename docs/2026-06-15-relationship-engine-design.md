# Molly 关系引擎 —— 设计方案（v1，2026-06-15）

> 起因：Kevin 指出 5 个问题——① 本命高亮的意义 ② 深度解读永远一样 ③ 预设问题不够懂这个人 ④ 对话答完不给下一个问题→跳出率高 ⑤ 合盘 UI/UX 粗糙。
>
> **诊断：5 个症状，一个病。** Molly 承诺「一个比你自己更懂你的人」，但产品是**参数表**不是**关系**。关系缺三样：**记忆（越来越懂你）、主动（牵着你走）、specificity（说的是你，不是你的星座）**。本文给出补齐这三样的完整方案：一个 **UserModel 记忆回路** 作为地基，4 个出口渲染它，外加合盘独立重做。

---

## 第一原则：抓住人性（凌驾一切设计决策）

**方向必须符合人性、深入人性、抓住人性。** 以下六条是它的可执行展开，也是对 v2 mockup 的人性碰撞修正：

1. **狠，必须被"赚来"。** 人只能从"确定站他这边"的人嘴里听进狠话；狠 without 盟友感 = 被审判 → 防御 → 逃。所以 **狠度 = f(懂你度/信任)**：Day-1 温暖而精准，随信任上升才敢狠。狠和深一样靠关系解锁——复用 keystone。
2. **权威外包给星盘，Molly 当盟友不当审判者。** "我看穿你"(presumptuous) → "这是你生来就带的"(宇宙在描述你)。她和你一起读盘，不拿盘压你。一字之差，接受度天差地别。
3. **追问给"掌控感"，不逼到墙角。** 三个追问别都往下挖——一个更深、一个"这对现在的我意味着什么"、一个"我能拿它怎么办"。人感到"我在开车"时留，被逼到角落时逃。
4. **深度的门是"认知诚实"，不是 usage gate。** "聊几次才解锁"(交易感、反懂你) → "我还不够懂你，不敢乱说"。门变成诚信标志，不是付费墙变体——接住反撒谎价值。
5. **日活靠"有人在等你"，不是 streak 威胁。** 为"被理解"的产品造 FOMO，思考型用户会察觉被操纵。治孤独的产品，最强返回是"有人记得你/今早想到你"。streak 走"我们一起走过 N 天"的暖框，不是 Snapchat 威胁。
6. **抓人性是"服务"不是"剥削"（底线铁律）。** 本品戳依恋伤/孤独/焦虑。**永不把用户留在伤口里，永远落到能动性 + 自我慈悲的转身。** 养着伤口换 engagement = 慢性自杀，没人能在"用完更难受"上建立日常关系。

> 这六条把 keystone 推得更中心：**狠度、深度、解锁全成了"赚来的信任"的函数**。第 6 条是新增伦理护栏，凌驾留存指标。

## 1. Keystone：UserModel —— Molly 对你的真实记忆

一个**持久、累积、会衰减、可回喂**的用户模型，纯本地（用户自己的数据，符合 app 既有隐私立场）。

### 1.1 输入信号（全部已在采集）

| 信号 | 来源 | 性质 |
|---|---|---|
| 心情日志 | `molly_moodlog_<date>` | 行为，**无需 NLP** |
| 每日命中 | `molly_verdict_<date>` + store `dailyHits/Misses/checkinDays` | 行为，**无需 NLP**，是「懂你度」的真实底 |
| 对话历史 | chat msgs | 需轻量抽取 |
| 主题浏览 | `track("theme_view")` | 行为 |
| 本命盘 | store `chart` | 固定真相 |

> **设计稳健性自查**：最强信号（心情、命中）**不需要 NLP**。即使用户几乎不聊天，模型靠心情+命中+盘也已远胜现状。对话抽取是锦上添花，不是地基的承重点——**模型对稀疏对话优雅降级**。

### 1.2 数据结构

```ts
// lib/model/userModel.ts
export interface UserModel {
  updatedAt: number;
  interactionCount: number;
  daysActive: number;

  // 情绪基线 —— 从心情日志聚合（纯行为，无 NLP）
  mood: {
    dominant: string | null;        // 出现最多的心情
    volatility: number;             // 0..1，摆动幅度
    recentTrend: "down" | "up" | "flat";  // 近 7 天
    lowStreak: number;              // 连续低落天数
  };

  // 校准信任 —— 从 verdict 聚合，是「懂你度」的真实底
  trust: {
    hits: number;
    misses: number;
    hitRate: number;                // hits/(hits+misses)
  };

  // 生活上下文 —— 从对话抽取（唯一需要 NLP 的部分；可缺省）
  themes: Array<{
    topic: string;                  // 感情/工作/家庭/自我/金钱/健康
    weight: number;                 // 投入度（recency 加权）
    lastSeen: number;
    facts: string[];                // 具体事实："刚分手"、"在考虑换工作"
  }>;

  // 开放回路 —— 该回访追问的事
  openLoops: Array<{ text: string; raisedAt: number }>;
                                    // "你说要和妈妈谈谈，谈了吗？"
}
```

### 1.3 累积 + 衰减

- 每次交互增量更新；**recency 加权**：近 7 天信号权重高，旧生活上下文按半衰期衰减（6 个月前的分手权重很低）。
- `themes[].weight *= decay^(daysSince/halfLife)`，低于阈值剔除。
- 纯函数 + 本地持久化（`molly_usermodel`），可单测，先红后绿。

### 1.4 把假「懂你度」做成真的（兑现承诺、接上反撒谎纪律）

```
understanding = clamp( base
  + W1 * trust.hitRate          // 她说中了吗（核心）
  + W2 * coverage(themes)        // 聊到的人生面有多广
  + W3 * recencyDepth )          // 近期投入深度
```

**关键诚实点**：用户标 miss → `hitRate` 降 → 数字**会停、会跌**。它不再单调装饰，而是**真实校准信号**。这正是本 session 反撒谎纪律在产品层的延伸：进度条不许撒谎。

> 机制自查：必须能跌，否则仍是假的。设计已保证（hitRate 进入分子，miss 直接拉低）。

## 2. 四个出口：渲染记忆

### #1 本命高亮 —— 从「标签」到「第一眼看见你」
- 框成「**我第一眼在你身上看到的三件事**」（关系开场白，非参数表）。
- 正文用**情绪语言**，星位降为**小字旁注**：
  正文「你爱起来从不浅尝——要么不碰，要么掏空自己。」· 旁注「金星天蝎·七宫」
- **可点** → 个性化微解读 / 直接挑起对话（"想知道为什么吗 →"）。
- 副标题可吃 UserModel：「你最近老是低落——这颗土星就是为什么。」

### #2 深度解读 —— 字面意义上"每次更深一层"
- 入参 `(chart, themeId)` → `(chart, themeId, userModel)`。
- **事实层稳定**（盘不变），**关系层生长**：有生活上下文时开头改写——
  「上次你说你们分开了——你金星天蝎，这就是为什么对你放手比别人难十倍。」
- "更深一层" **由 understanding 解锁，不是付费墙**——把现在的 `🔒 即将开放` 占位换成"她越懂你，解锁越深"。
- 进页显示"自上次以来新增了什么"。

### #3 预设问题 —— 从「星盘镜头」到「她的人生」
- 生成入参 `(chart × userModel)`：
  - **零上下文**：把"话题"改"伤口"——「为什么我总在快得到时亲手搞砸?」≫「聊聊我的感情」。
  - **有上下文**：指向真实信号 + openLoops——「你妈那事后来怎样了?」
- chat 三个写死开场白（`chat/page.tsx:19`）必须至少按盘个性化。

### #4 对话后续问题 —— 最大留存杠杆（可不依赖记忆先上）
- 每条 Molly 回复**下面**挂 2-3 个后续，要求：
  (a) 从她**刚说的**长出来 (b) **逐级加深**（往伤口里走） (c) 用她口吻 (d) 含一个"她不敢问但想知道"的。
- **人性机制**：情绪敞开的当口让她自己想问题=高认知+情绪负荷→她带着"被看穿"的高峰**关掉 app**（peak-end 在高点离开）。好咨询师**永不在突破后留白**。后续问题=把她留在引力场的重力。
- **实现**：chat completion 同时返回 `reply` + `followups[]`（结构化），或一次极快的二次调用；渲染为每条消息下的 chips。注意延迟——优先同一次调用返回。

## 3. 合盘重做（#5，独立正交轨）

### 3.1 分型解读（杀掉 mad-lib）
- 现 `reading()`（`synastry/page.tsx:23-32`）5 类型共用模板，只填最高/最低维度，且金句「得有一个人先松口」对同事/朋友盘是错的。
- 改：**每个 RelType（lover/partner/colleague/friend/family）独立 copy 银行 / 生成**，语气与维度命名都按类型走。

### 3.2 UI/UX 补完
- 结果**点名两个人**：「你 ↔ 小鱼 · 恋人盘」（现在结果里两人都没名字）。
- **等待态**：显示对方名字 + 倒计/脉冲，不再静默 4s 轮询。
- **显眼的"发链接给 TA"**：现在 A 根本不知道怎么把 invite 发出去——病毒回路第一步就断。做成明确分享动作。
- **维度可点** → "哪些星位造成的"（用 synastry 的 aspect 数据下钻）。
- **"再合一个人"循环** + 管理已合的人。
- 把它当**全 app 最病毒的面**经营（关系 + 分享天然传播）。

## 4. 分期落地建议

| 阶段 | 内容 | 依赖 | 成本 | 价值 |
|---|---|---|---|---|
| **P0 先上** | #4 对话后续问题 | 无（用现成对话上下文） | 低 | 留存最大杠杆 |
| **P1 地基** | UserModel：信号聚合 + 诚实 understanding | P0 无关 | 高 | 1/2/3 的地基；假进度条变真 |
| **P2 回喂** | #3 问题 → #2 解读深化 → #1 高亮 hook | P1 | 中 | 同一引擎四出口 |
| **P3 增强** | 对话→生活上下文 抽取引擎 | P1 | 中 | 让一切更厚 |
| **并行轨** | #5 合盘 depth + polish | 无 | 中 | 病毒面 |

每阶段独立可测、独立上线，沿用 worktree 隔离 + 先红后绿 + 分档诚实汇报 + 出屏 render 验证。

## 5. 风险 / 机制自查

1. **对话抽取稀疏** → 模型靠心情+命中+盘优雅降级（已设计）。
2. **understanding 必须能跌** → hitRate 进分子，miss 拉低（已保证），否则仍是假。
3. **后续问题延迟** → 优先同一次 completion 返回，避免二次调用卡顿。
4. **隐私** → 全本地、用户自有数据，符合既有立场。
5. **分型解读维护成本** → 类型 copy 银行集中在一个数据文件，避免散落组件。

## 6. 装桌面（A2HS）· 环境感知矩阵（v2 修订）

**关键纠正：A2HS 能否成功取决于「浏览器环境」，不是「OS」。** 纯按 iOS/Android 分页面会在微信 webview、iOS-Chrome 等处翻车。检测维度 = 环境 × 引擎（UA 字符串 + `beforeinstallprompt` 是否可用）。

| 环境 | 检测 | A2HS | 提示 |
|---|---|---|---|
| iOS Safari | iOS 且无 CriOS/FxiOS/EdgiOS | ✅ 手动 | 底部分享键(□↑)教程 |
| **iOS Chrome/Edge** | UA 含 **CriOS / EdgiOS** | ⚠️ ⋯菜单可加，未必 standalone | 走 Chrome ⋯ 菜单两步 + 软引导转 Safari |
| iOS 微信/小红书/微博/QQ | UA 含 **MicroMessenger / WeiBo / XHS / QQ** | ❌ | 转 Safari 打开 |
| Android Chrome · BIP 触发 | `deferred != null` | ✅ 一键 | 原生一键按钮 |
| Android Chrome · 无 BIP | Android+Chrome 无 deferred | ✅ 菜单 | ⋮ → 安装应用 |
| Android 其他浏览器 | Samsung/UC/小米… | ⚠️ 菜单 | ⋮ → 添加到主屏幕 |
| Android 微信等 webview | UA 含 `wv` / MicroMessenger | ❌ | 用浏览器打开 |
| 桌面 Chrome/Edge | desktop + BIP | ✅ | 地址栏安装图标 |
| 已 standalone | display-mode:standalone | — | 不显示 |

检测字符串：`CriOS`=iOS Chrome、`EdgiOS`=iOS Edge、`FxiOS`=iOS Firefox、`MicroMessenger`=微信、`wv`=Android webview。**纯函数 + 先红后绿单测**（喂各种真实 UA，断言落到哪一态）。

## 7. 文案语气基调（v2 决定）

Molly 文案**更狠**：少缓冲、直接命名伤口、带钝痛感（"你不是累，你是撑了太久，久到忘了那叫撑"）。仍守 persona 安全栏（不医疗/不绝对、危机走共情+资源）。后续问题里至少一个是"她不敢自己问"的那个。合盘金句同样走狠路线（"不会死于不爱，会死于都太硬"）。
