## P2 — polish / a11y (selected; full lists in agent outputs)
- **Tap targets <44px:** chat & reading send ➤ (42), BackButton (~28), share template thumbs (36×48), wealth cells (~40). TabBar is correct (44).
- **`window.prompt()` for nickname edit** (settings) — jarring native dialog, breaks the dark H5 feel, no validation; same for `window.confirm` on delete.
- **Synastry relationship icons are emoji** (💞🤝💼👯👩‍👧) — inconsistent with the custom line-icons in the TabBar.
- **`role="checkbox"` without accessible name** (me/birth, invite) — SR reads "checkbox" with no label.
- **chat `aria-live` wraps the whole transcript** — re-announces history on each reply.
- **theme deep-link/refresh** renders a brief blank screen (`if(!r) return null`, no skeleton).
- **invite headline flips「有人」→ real name** with no loading state.
- **nickname / email no truncation** — long values overflow the header/row.
- **No `<form>` on input** (no native Enter-submit); **city error is color-only** (no `aria-invalid`).
- **LoadingRitual** has no `role="status"` (silent for SR during the wait).
- **focus ring** hardcodes `border-radius:6px` so it's square-ish on round buttons.

## The 3 highest-value
1. **F-1 form overflow** — a submit button you can't reach is the closest thing to a blocker here.
2. **CON-1 contrast** — systemic readability; the product's core value is *text* and some of it is below AA.
3. **R-1 / SET-3 / SHR-1 honesty trio** — controls that look functional but aren't (fake composer, dead toggles, same-action channel buttons).
