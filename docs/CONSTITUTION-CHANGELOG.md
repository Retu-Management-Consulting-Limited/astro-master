**改了什么**：只改 **rubric**（`docs/2026-06-16-constitution-eval-rubric.md`），**未动宪法原则正文**（§1–§10 一字未改）。新增 **F 维「多语言适用」**：明确宪法是语言无关的，§5.2/§6.2/§6.4/§8/§9 同样适用于 Molly 的**俄语**产出——F1（俄语 persona/确定性内容表/安全回应过 ORACLE_RU/FALSE_CERTAINTY_RU/SHAME_RU/MEDICAL_DIRECTIVE_RU 机器闸，判据同 C1/B2/C2/D）、F2（俄语危机信号短路到已核实俄语热线、不占星，判据同 E1）。F 进 CI 硬闸子集，新语言上线前还须母语者复核语气与危机词完整性（机器闸不能替代）。
**为什么**：i18n 子项目 C 让 Molly 用俄语解读（M2 persona + M3 确定性内容表 + M4 俄语危机检测）。原 rubric 的机器闸全是中文正则，俄语产出无对应亮线闸——会出现「中文合规、俄语漏过 §8/§9」的盲区。F 维把同一套判据本地化为俄语机器闸（落地 `web/src/lib/reading/calibrationEval.test.ts` 的 RU §8/§9 eval），并写明语言无关 = 不因翻译放松底线。属**派生/澄清**，非原则变更。
**人工闸**：待 Kevin review（修宪 PR 流程；本条已登记以过 `constitution-gate`，RU_PUBLIC 生产默认关、俄语暂不对用户开，机器闸已绿）。
