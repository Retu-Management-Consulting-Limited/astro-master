# Full-app bug audit (2026-06-14)

Swept every page/component for the class of bug found on /today: **dead controls,
fake-personal hardcoded content, frozen dates, dead-end inputs, unbacked promises.**
All findings verified against source by 5 parallel auditors. Fixes NOT applied — this
is the list.

> Context: `NEXT_PUBLIC_MOLLY_AI=1` in this checkout, so `/reading`, `/theme`, `/chat`
> live replies DO call real AI over the real chart. Several "fake-personal" items are
> the **stub/fallback** content (shown ~1.4s, or whenever AI is off/fails), and the
> **non-AI** screens (today/chart/wealth daily copy) which never call AI at all.

## 根因聚类（这些 bug 其实来自 5 个根）

- **根 A · 内容写死冒充个性化**（FAKE-PERSONAL）：TD-3 / CH-1 / CT-1 / CT-2 / ME-1 / ME-3 / W-4 / RD-2。多屏直接 ship 硬编码占星文案，当作「你的」解读。**最严重的是 CT-1「她记得」和 ME-1「我眼中的你」——招牌钩子是假的。**
- **根 B · 日期冻结**（FROZEN-DATE）：TD-2 / W-1 / W-2。全站没有一处用真实「今天」；today 与 wealth 钉死在 2026-06-13/六月，**已经过期且永不前进**。
- **根 C · 死控件**（DEAD-CONTROL）：TD-1 / ME-2 / RD-1 / SH-1。看着可点、点了没做它表示的事。
- **根 D · 输入黑洞**（DEAD-END-INPUT）：TD-4 心情 / SET-1 通知 / CAL-1 校准。用户给了信号，没有任何下游消费它。
- **根 E · 没兑现的承诺**（UNBACKED-PROMISE）：TD-1/TD-6「+1%校准」、CH-2「越用越亮」、W-3 + SET-1「提醒你」、ME-2「看完整」、CAL「越看清你」。UI 说会发生的事，没代码兑现。

> 这正是已写进 DESIGN-SYSTEM 的 **R2（占位值不许变字面量）** 和 **R4（宣称必须同 PR 兑现或撤回）** 的批量历史违例——规则是后立的，这些屏在立规前就 ship 了。

## 一句话
**「今日」不是个案。`/chart`、`/chat`、`/me`、`/wealth`、`/calibration` 都有同型 bug——核心是：很多屏是『写死内容 + 冻结日期 + 死控件 + 落空承诺』的样板，冒充活的个性化产品。** 真正端到端为真的：/synastry、/synastry/invite、/me/birth、/history、/share（卡片本体）、注册/登录/删除/导出、FeedbackButton 管线、PWA 离线壳。
